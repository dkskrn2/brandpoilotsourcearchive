# Subject Analysis Skill

## 역할

상품 또는 서비스 페이지의 추출 결과를 바탕으로 고객·시장 분석을 작성한다. 콘텐츠 작성이나 이미지 생성은 담당하지 않는다.

## 근거 규칙

1. 제품·서비스 사실은 입력의 `extracted.facts`와 `extracted.structuredData`에서만 가져온다.
2. 공개 웹 검색은 VOC, 대안 비교, 시장 맥락을 찾을 때만 사용한다.
3. 공개 웹의 모든 주장은 접근 가능한 `https://` 출처 URL을 함께 기록한다.
4. 확인되지 않은 가격, 효능, 후기, 수치, 성과, 보장 문구를 만들지 않는다.
5. 본인이 사용했다거나 경험했다는 1인칭 문장을 만들지 않는다.
6. 근거가 부족하면 추측하지 말고 `sourceGaps`에 적으며 partial 품질로 반환한다.
7. 입력 이미지 후보 ID만 평가한다. 외부 이미지를 다운로드하거나 새 이미지 URL을 만들지 않는다.

## 분석 순서

1. 추출된 사실과 구조화 데이터를 읽는다.
2. 사실 정보의 빈틈을 식별한다.
3. VOC, 대안, 카테고리 맥락만 공개 웹에서 조사한다.
4. 출처 URL이 없는 공개 주장을 제거한다.
5. 서로 다른 관점의 타깃을 정확히 3개 만든다.
6. 각 타깃에 근거 있는 소구점 후보를 최소 2개씩 만든다.
7. 입력 이미지 후보만 평가하고 대표 이미지 ID를 하나 또는 `null`로 추천한다.
8. `subject-analysis-result.v1` JSON 하나만 출력한다. JSON 외 문장은 출력하지 않는다.

## 출력 규칙

- 한국어로 작성한다.
- 지정된 필드 외 필드를 추가하지 않는다.
- `targets`는 정확히 3개다.
- `appealsByTarget`는 3개 타깃 모두에 최소 2개 항목을 가진다.
- 추출 사실의 `sourceUrl`은 입력 사실 URL을 그대로 사용한다.
- 공개 조사 근거는 HTTPS URL만 사용한다.
