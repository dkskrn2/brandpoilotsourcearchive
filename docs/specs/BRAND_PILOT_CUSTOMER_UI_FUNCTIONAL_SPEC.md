# Brand Pilot Customer UI Functional Spec

작성일: 2026-07-05  
상태: Draft, 고객용 IA 확정 기반 상세 기능 명세  
대상: 고객용 웹앱 MVP  

## 1. 목적

이 문서는 Brand Pilot Managed Content Automation MVP의 고객용 화면별 상세 기능을 정의한다.

상위 제품 결정은 `BRAND_PILOT_MANAGED_CONTENT_AUTOMATION_MVP.md`를 따른다. 이 문서는 UI 구현 전에 필요한 화면 단위 기능, 버튼, 상태값, 빈 화면, 오류 화면, 자동 승인 차단 사유를 고정하는 것을 목적으로 한다.

디자인 시스템 기준은 `BRAND_PILOT_CUSTOMER_UI_DESIGN_SYSTEM.md`를 따른다.

화면 와이어프레임은 `BRAND_PILOT_CUSTOMER_UI_WIREFRAMES.md`를 따른다.

## 2. 범위

포함:

- 온보딩
- 콘텐츠
- 게시 큐
- 소스
- 채널
- 브랜드 설정
- 자동 승인 상태와 차단 사유 표시
- 연결 오류/권한 오류 표시

제외:

- 운영자 전용 화면
- 대시보드
- 콘텐츠 직접 편집기
- 사용자가 게시 시간을 직접 선택하는 기능
- 사용자가 채널별 하루 생성/게시 개수를 설정하는 기능
- 결제/요금제
- 성과 분석
- 댓글/DM 관리

```text
온보딩

콘텐츠
  - 검토 필요
  - 자동 승인됨
  - 자동 승인 차단
  - 재생성 중
  - 거절됨

게시 큐
  - 오늘 게시 예정
  - 예정
  - 게시 완료
  - 실패

소스
  - 자사 URL
  - 참고 URL
  - 주제표 업로드
  - 주제 큐

채널
  - Instagram
  - Threads
  - Webflow
  - 연결 오류/권한 상태

브랜드 설정
  - 브랜드 프로필
  - 톤앤매너
  - 금지 표현
  - CTA
  - 기본 링크
  - 전체 자동 승인 on/off
```

기본 진입 규칙:

1. 온보딩 미완료 상태면 `온보딩`으로 진입한다.
2. 검토 대기 콘텐츠가 있으면 `콘텐츠 > 검토 필요`로 진입한다.
3. 오늘 게시 예정 또는 실패 항목이 있으면 `게시 큐`에서 확인할 수 있어야 한다.
4. 설정이 완료되고 검토할 콘텐츠가 없으면 `게시 큐 > 오늘 게시 예정`을 기본 업무 화면으로 둔다.

## 4. 공통 UI 규칙

디자인 토큰, 공통 컴포넌트, 상태 배지, 레이아웃 패턴은 `BRAND_PILOT_CUSTOMER_UI_DESIGN_SYSTEM.md`를 기준으로 한다.

### 4.0 프론트엔드 스택

고객용 웹앱은 React 기반으로 구현한다.

기본 스택:

- React
- TypeScript
- Next.js App Router
- Tailwind CSS
- shadcn/ui
- Radix UI primitives
- React Hook Form
- Zod

선택 이유:

- `ui-styling` 스킬과 가장 잘 맞는 조합이다.
- shadcn/ui를 사용하면 버튼, 폼, 탭, 모달, 테이블, 배지 같은 운영툴 컴포넌트를 빠르게 구성할 수 있다.
- Tailwind CSS로 상태 배지, 검토함, 게시 큐, 채널 연결 상태를 일관된 토큰으로 관리하기 쉽다.
- Next.js App Router는 고객용 화면, 인증 후 페이지, OAuth callback, API route를 같은 React 기반 프로젝트 안에서 다루기 좋다.

구현 원칙:

- 화면은 React Server Component보다 클라이언트 상호작용이 많은 운영툴로 본다.
- 데이터 입력 폼은 React Hook Form과 Zod로 검증한다.
- 공통 컴포넌트는 `components/ui`와 `components/app`을 분리한다.
- shadcn/ui 컴포넌트는 그대로 쓰되, Brand Pilot 전용 상태 배지와 업무 컴포넌트는 별도 래핑한다.

### 4.1 공통 레이아웃

- 좌측 내비게이션을 기본 구조로 사용한다.
- 상단에는 현재 브랜드명, 연결 상태 요약, 마지막 생성 시간을 표시한다.
- 대시보드는 만들지 않는다.
- 같은 고객이 여러 브랜드를 운영할 가능성은 데이터 모델에서 열어두되, MVP UI는 단일 브랜드 기준으로 시작한다.

### 4.2 공통 액션 규칙

- 위험한 액션은 확인 모달을 띄운다.
- 삭제는 즉시 hard delete가 아니라 MVP에서는 `disabled` 또는 `inactive` 처리한다.
- 저장 버튼은 변경 사항이 있을 때만 활성화한다.
- 네트워크 요청 중에는 버튼을 disabled 처리하고 로딩 상태를 표시한다.
- 실패한 액션은 사용자에게 실패 이유와 다음 행동을 보여준다.

### 4.3 공통 상태 배지

연결 상태:

- `not_connected`: 연결 전
- `connected`: 정상 연결
- `needs_attention`: 확인 필요
- `expired`: 토큰 만료
- `insufficient_permissions`: 권한 부족
- `mapping_required`: 필드 매핑 필요
- `publish_failed`: 최근 게시 실패

검토 상태:

- `pending_review`: 검토 필요
- `approved`: 수동 승인됨
- `auto_approved`: 자동 승인됨
- `auto_approval_blocked`: 자동 승인 차단
- `rejected`: 거절됨
- `regenerating`: 재생성 중
- `regenerated`: 재생성 완료

게시 상태:

- `queued`: 큐에 있음
- `scheduled`: 게시 예정
- `publishing`: 게시 중
- `published`: 게시 완료
- `failed`: 게시 실패
- `deferred`: 다음날 이월

소스 상태:

- `active`: 사용 중
- `crawling`: 크롤링 중
- `crawled`: 크롤링 완료
- `crawl_failed`: 크롤링 실패
- `disabled`: 비활성화

주제 행 상태:

- `uploaded`: 업로드됨
- `queued`: 생성 후보
- `used`: 사용 완료
- `skipped`: 스킵됨
- `invalid`: 유효하지 않음
- `failed`: 사용 실패

## 5. 온보딩

### 5.1 목적

고객이 콘텐츠 생성과 자동 게시를 시작하기 위해 필요한 최소 조건을 확인한다.

### 5.2 표시 정보

체크리스트 항목:

- 브랜드 정보 입력
- 자사 URL 등록
- 참고 URL 등록
- 주제표 업로드
- Instagram 연결
- Threads 연결
- Webflow 연결
- 첫 콘텐츠 생성
- 첫 승인 또는 자동 승인
- 첫 게시 완료

각 항목 표시:

- 항목명
- 설명
- 상태
- 마지막 업데이트 시간
- 이동 버튼
- 오류가 있으면 오류 요약

### 5.3 버튼

- `브랜드 설정하기`
- `URL 추가하기`
- `주제표 업로드`
- `채널 연결하기`
- `첫 콘텐츠 생성 요청`
- `게시 큐 보기`

### 5.4 상태값

- `complete`: 완료
- `required`: 필요
- `in_progress`: 진행 중
- `error`: 오류
- `needs_attention`: 확인 필요

### 5.5 빈 화면

아직 브랜드가 없으면:

- 브랜드 설정 시작 버튼을 보여준다.
- 다른 화면으로 이동하더라도 콘텐츠 생성 관련 액션은 비활성화한다.

### 5.6 오류 화면

대표 오류:

- 브랜드 필수 정보 누락
- 주제표 유효 행 없음
- 채널 연결 실패
- Webflow 필드 매핑 미완료
- Meta 권한 부족

오류 화면은 문제, 원인, 다음 액션을 함께 보여준다.

## 6. 콘텐츠

### 6.1 목적

생성된 채널별 콘텐츠를 검토하고, 승인/재생성/거절 또는 자동 승인 상태를 확인한다.

### 6.2 하위 탭

- 검토 필요
- 자동 승인됨
- 자동 승인 차단
- 재생성 중
- 거절됨

### 6.3 리스트 표시 정보

콘텐츠 카드 또는 테이블 컬럼:

- 주제 제목
- 채널
- 생성 시간
- 사용된 소스
- 검토 상태
- 자동 승인 상태
- 게시 큐 상태
- 예정 게시 시간
- 최근 오류 또는 차단 사유

필터:

- 채널
- 상태
- 생성일
- 주제표 업로드 파일
- 소스 URL

정렬:

- 검토 필요 우선
- 게시 예정 임박순
- 생성 최신순
- 오류 우선

### 6.4 콘텐츠 상세

상세 화면 구성:

- 주제 정보
- 생성 근거
- 채널 탭: Instagram / Threads / Webflow
- 미리보기
- 자동 승인 검사 결과
- 게시 큐 상태
- 재생성 이력

채널별 미리보기:

Instagram:

- 카드뉴스 슬라이드 미리보기
- 정방형 1:1 슬라이드 기준
- 기본 렌더링 크기 1080 x 1080
- 캡션
- 해시태그
- CTA
- 이미지 렌더링 상태

Threads:

- 본문
- 스레드 구조
- 링크/CTA
- 글자 수

Webflow:

- 제목
- slug
- 요약
- 본문 미리보기
- 메타 제목
- 메타 설명
- 대표 이미지 상태
- Webflow 필드 매핑 상태

### 6.5 버튼

검토 필요 상태:

- `승인`
- `재생성`
- `거절`
- `게시 큐 보기`

자동 승인됨 상태:

- `게시 큐 보기`
- `자동 승인 검사 보기`
- `거절`

자동 승인 차단 상태:

- `수동 승인`
- `재생성`
- `거절`
- `차단 사유 보기`

재생성 중 상태:

- `재생성 상태 새로고침`

거절됨 상태:

- `상세 보기`
- `같은 주제로 다시 생성`

### 6.6 액션 규칙

승인:

- 채널별 결과물을 승인한다.
- 승인 즉시 게시 큐에 `queued` 상태로 들어간다.
- 이미 게시 큐에 있으면 중복 등록하지 않는다.

재생성:

- 재생성 사유 입력을 받는다.
- 직접 편집은 제공하지 않는다.
- 기존 결과물은 이력으로 남긴다.
- 상태는 `regenerating`으로 바뀐다.

거절:

- 결과물을 게시하지 않는다.
- 거절 사유는 선택 입력이다.
- 거절 후에도 이력 확인은 가능하다.

자동 승인됨:

- 사용자가 누르는 버튼이 아니라 시스템이 정책 통과 시 부여하는 상태다.
- 게시 전까지 게시 큐에서 확인 가능해야 한다.

### 6.7 빈 화면

검토 필요:

- "검토할 콘텐츠가 없습니다."
- 다음 생성 예정 시간과 필요한 설정 상태를 보여준다.

자동 승인됨:

- "자동 승인된 콘텐츠가 없습니다."
- 브랜드 설정으로 이동하는 버튼을 보여준다.

자동 승인 차단:

- "자동 승인 차단 항목이 없습니다."
- 차단 항목이 없다는 것은 오류가 아니라 정상 상태로 표시한다.

재생성 중:

- "재생성 중인 콘텐츠가 없습니다."

거절됨:

- "거절된 콘텐츠가 없습니다."

### 6.8 오류 화면

대표 오류:

- 콘텐츠 미리보기 로드 실패
- 이미지 렌더링 결과 없음
- Webflow 필드 매핑 누락
- 게시 큐 등록 실패
- 재생성 요청 실패
- 이미 게시된 콘텐츠를 거절하려는 경우

## 7. 자동 승인 차단 사유

자동 승인 차단 사유는 사용자가 바로 이해할 수 있는 문장으로 보여준다.

### 7.1 공통 차단 사유

- 금지 표현 포함
- 외부 참고 URL 문장 복제 위험
- 브랜드 정보와 불일치
- 필수 CTA 누락
- 필수 링크 누락
- 최근 콘텐츠와 주제 중복 가능성 높음
- 콘텐츠 길이 규격 초과
- 콘텐츠 길이 규격 미달
- 민감 표현 포함
- 가격, 법률, 의료성 표현 확인 필요
- 생성 근거 부족
- 브랜드 전체 자동 승인 꺼짐

### 7.2 Instagram 차단 사유

- 이미지 렌더링 실패
- 정방형 이미지 규격 오류
- 카드뉴스 슬라이드 수 규격 오류
- 이미지 공개 URL 접근 불가
- 캡션 길이 초과
- 해시태그 수 과다
- Instagram 계정 연결 오류
- Instagram 권한 부족
- Instagram 토큰 만료
- Professional 계정 조건 미충족

### 7.3 Threads 차단 사유

- 본문 길이 초과
- 스레드 분할 실패
- Threads 계정 연결 오류
- Threads 권한 부족
- Threads 토큰 만료
- 링크 형식 오류

### 7.4 Webflow 차단 사유

- Webflow 연결 오류
- Webflow API token 만료 또는 권한 부족
- Site ID 누락
- Collection ID 누락
- 필드 매핑 누락
- slug 중복
- Rich text 변환 실패
- 대표 이미지 누락 또는 업로드 실패

## 8. 게시 큐

### 8.1 목적

승인 또는 자동 승인된 콘텐츠가 언제 어느 채널에 게시되는지 보여준다.

### 8.2 하위 탭

- 오늘 게시 예정
- 예정
- 게시 완료
- 실패

### 8.3 표시 정보

게시 항목:

- 주제 제목
- 채널
- 승인 유형: 수동 승인 / 자동 승인
- 큐 상태
- 게시 예정 시간
- 실제 게시 시간
- 실패 사유
- 재시도 횟수
- 소스 콘텐츠 링크

채널별 슬롯:

- 11:30 기준 슬롯
- 14:30 기준 슬롯
- 17:30 기준 슬롯
- 20:30 기준 슬롯
- 각 슬롯은 정책 지터 적용 후 실제 예정 시간을 표시한다.

### 8.4 버튼

- `콘텐츠 보기`
- `게시 상태 새로고침`
- `실패 사유 보기`
- `채널 연결 확인`
- `수동 재시도 요청`

사용자가 직접 게시 시간을 변경하는 버튼은 제공하지 않는다.

### 8.5 상태값

- `scheduled`
- `publishing`
- `published`
- `failed`
- `deferred`

### 8.6 빈 화면

- 슬롯 계산 실패

## 15. 브랜드 전체 자동 승인

### 15.1 목적

브랜드 설정에서 전체 자동 승인 사용 여부와 차단 조건을 확인한다.

### 15.2 표시 정보

- 브랜드 전체 자동 승인 on/off
- 적용 범위
- 차단 조건 요약
- 최근 자동 승인 수
- 최근 자동 승인 차단 수

### 15.3 버튼

- `자동 승인 켜기`
- `자동 승인 끄기`
- `차단 조건 보기`
- `최근 자동 승인 콘텐츠 보기`
- `최근 차단 콘텐츠 보기`

### 15.4 기본 정책

- 브랜드 전체 자동 승인 기본값은 off다.
- on 상태에서는 연결된 모든 채널 결과물이 동일한 자동 승인 검사 대상이 된다.
- Instagram/Threads/Webflow 중 연결 또는 권한 조건이 부족한 채널 결과물은 자동 승인 차단 상태가 된다.
- 자동 승인된 콘텐츠도 게시 큐에 표시한다.

### 15.5 빈 화면

- "브랜드 전체 자동 승인이 꺼져 있습니다."
- 자동 승인은 선택 기능이며, 수동 승인만으로도 운영 가능하다는 점을 표시한다.

### 15.6 오류 화면

대표 오류:

- 채널 연결 전 자동 승인 활성화 시도
- 필드 매핑 미완료 상태에서 자동 승인 활성화 시도
- 권한 부족 상태에서 자동 승인 활성화 시도

## 19. 자체 검토

- 대시보드는 포함하지 않았다.
- 운영자 화면은 제외했다.
- 직접 편집 기능은 제외했다.
- 게시 시간 직접 설정 기능은 제외했다.
- 채널별 하루 생성/게시 개수 설정은 제외했다.
- 자동 승인은 브랜드 전체 설정으로 정의했다.
- 자동 승인 차단 사유는 공통/채널별로 분리했다.
- Webflow는 API token으로 시작하고 OAuth 전환을 열어두는 구조로 정의했다.
- Instagram/Threads는 우리 Meta 앱 기반 OAuth 연결 전제를 유지했다.
