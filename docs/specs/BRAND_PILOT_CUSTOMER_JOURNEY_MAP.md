# Brand Pilot Customer Journey Map

작성일: 2026-07-05  
상태: Draft  
범위: 고객용 UI 여정. 운영자 여정은 제외한다.

## 1. Actor

**초기 파일럿 고객: 콘텐츠 운영 부담이 큰 소규모 브랜드/사업자**

- 이미 사업, 상품, 서비스, 홈페이지, FAQ, 블로그, 고객 사례 같은 자료를 가지고 있다.
- Instagram, Threads, Webflow에 꾸준히 콘텐츠를 올리고 싶지만 직접 운영할 시간이 부족하다.
- 완전 무검수 자동 게시보다는 승인 또는 자동 승인 정책을 확인한 뒤 게시되는 흐름을 선호한다.
- 채널 연결 세팅은 감수할 수 있지만, 매일 콘텐츠를 직접 만들고 채널별로 다시 쓰는 일은 줄이고 싶다.

## 2. Scenario + Expectations

고객은 Brand Pilot에 브랜드 정보, 자사 URL, 참고 URL, 주제표를 등록하고, 매일 생성되는 콘텐츠를 승인하거나 자동 승인 상태로 운영해 Instagram 정방형 카드뉴스, Threads 텍스트, Webflow 글이 자동 게시되길 기대한다.

기대:

- 처음 시작할 때 무엇을 입력해야 하는지 명확해야 한다.
- 콘텐츠가 어떤 근거로 생성됐는지 확인할 수 있어야 한다.
- 직접 편집하지 않아도 승인, 재생성, 거절로 충분히 운영 가능해야 한다.
- 자동 승인된 콘텐츠와 자동 승인 차단 콘텐츠를 구분할 수 있어야 한다.
- 게시 실패가 숨겨지지 않고, 원인과 다음 행동이 보여야 한다.

## 5. Phase Opportunities

### 5.1 시작 준비

기회:

- 온보딩 화면은 단순한 안내가 아니라 "현재 게시 가능 여부"를 바로 판단하는 상태판이어야 한다.
- 첫 생성 요청 버튼은 필수 조건이 충족되기 전까지 비활성화하고, 비활성 사유를 옆에 보여준다.
- 대시보드는 제외했으므로 온보딩이 초기 진입의 상태 요약 역할을 해야 한다.

측정:

- 온보딩 시작 후 첫 콘텐츠 생성 요청까지 걸린 시간
- 필수 항목 오류로 인한 이탈률

### 5.2 브랜드/소스 입력

기회:

- 자사 URL과 참고 URL의 역할을 UI에서 분명히 나눠야 한다.
- 주제표 업로드는 성공/실패만 보여주면 부족하다. 총 행, 유효 행, 중복 행, 필수값 누락 행을 바로 보여줘야 한다.
- 주제 행이 한 번만 사용된다는 정책을 주제 큐 화면에서 반복적으로 드러낸다.

측정:

- 유효 주제 비율
- URL 크롤링 성공률
- 주제표 재업로드 횟수

### 5.3 채널 연결

기회:

- 고객에게 개발자 콘솔 작업을 요구하지 않는 방향을 유지한다.
- Instagram/Threads는 우리 Meta 앱 기반 OAuth 연결을 전제로 두되, UI에는 권한 부족, 토큰 만료, 계정 타입 문제를 사람이 이해할 수 있는 문장으로 보여준다.
- Webflow는 필드 매핑이 핵심 실패 지점이므로, 연결됨과 게시 가능을 분리해서 보여준다.

측정:

- 채널 연결 성공률
- 권한/토큰 오류 발생률
- Webflow 매핑 완료율

### 5.4 콘텐츠 생성 대기

기회:

- 매일 오전 10시 생성 정책을 명확히 보여준다.
- 생성 결과가 없을 때 "아직 없음"인지 "생성 실패"인지 구분한다.
- 주제표가 부족하면 다음 생성이 멈춘다는 사실을 미리 알려야 한다.

측정:

- 생성 배치 성공률
- 생성 실패 후 복구 시간
- 사용 가능한 주제 잔량

### 5.5 검토/자동 승인

기회:

- 콘텐츠 검토 화면은 제품의 핵심 화면이다. 사용자는 여기서 "이걸 올려도 되는가"를 판단한다.
- Instagram은 정방형 1:1 카드뉴스 미리보기를 기본으로 보여준다.
- 자동 승인은 고객의 시간을 줄이는 기능이지만, 신뢰가 없으면 위험하게 느껴진다. 자동 승인 검사 결과와 차단 사유가 반드시 보여야 한다.
- 직접 편집을 제공하지 않기 때문에 재생성 사유 입력이 충분히 좋아야 한다.

측정:

- 수동 승인률
- 자동 승인률
- 자동 승인 차단률
- 재생성 후 승인률

### 5.6 게시 확인/복구

기회:

- 게시 큐는 캘린더 편집기가 아니라 운영 신뢰 화면이다.
- 사용자가 시간을 바꾸지는 않지만, 왜 비어 있는지, 왜 이월됐는지, 왜 실패했는지는 알아야 한다.
- 실패는 기술 오류 코드보다 복구 액션 중심으로 보여준다.

측정:

- 게시 성공률
- 실패 콘텐츠 재시도 성공률
- 실패 후 사용자가 채널 설정으로 이동한 비율

## 7. Critical Moments

### Moment 1. 첫 온보딩 완료

사용자가 "이제 생성이 가능하다"고 느껴야 한다. 체크리스트가 단순 목록이면 부족하고, 생성 가능/불가능 상태를 판정해야 한다.

### Moment 2. 첫 콘텐츠 검토

첫 결과물이 기대보다 낮으면 제품 신뢰가 급격히 떨어진다. 생성 근거, 채널별 미리보기, 재생성 사유 입력이 중요하다.

### Moment 3. 첫 자동 게시 성공

사용자가 직접 복사하지 않아도 게시가 완료되는 순간이 핵심 가치다. 게시 완료 시간과 채널 결과를 명확히 보여준다.

### Moment 4. 첫 게시 실패

실패를 숨기면 신뢰가 깨진다. 실패 원인, 영향 범위, 다음 액션을 한 화면에서 보여줘야 한다.

## 8. UX Principles From Journey

1. 상태를 숨기지 않는다. 생성, 승인, 게시, 실패 상태는 항상 노출한다.
2. 자동화는 통제감과 함께 제공한다. 자동 승인에는 검사 결과와 차단 사유가 필요하다.
3. 고객은 설정을 많이 하고 싶지 않다. 게시 시간과 채널별 개수 설정은 노출하지 않는다.
4. 콘텐츠 직접 편집 대신 재생성 사유 입력을 강화한다.
5. Instagram은 정방형 카드뉴스 경험으로 다룬다. 세로 피드/릴스 프리뷰처럼 보이면 안 된다.

## 9. Open Questions

1. 첫 버전에서 Webflow 자동 승인만 허용하고 Instagram/Threads는 수동 승인으로 고정할지 결정해야 한다.
2. 채널 연결 오류를 고객이 직접 해결할 수 있는 수준까지 안내할지, 운영자 확인 필요 상태로 넘길지 결정해야 한다.
3. 콘텐츠 생성 실패 알림을 사이트 내 표시만 할지, 이메일/메신저 알림까지 포함할지 결정해야 한다.
