## 1. 목적

이 문서는 Brand Pilot 고객용 MVP 화면의 low-fi 와이어프레임을 정의한다.

목표는 시각 디자인을 완성하는 것이 아니라, 화면 구조, 정보 우선순위, shadcn/ui 컴포넌트 매핑, 반응형 동작을 구현 전에 고정하는 것이다.

## 2. 전체 레이아웃

### 2.1 Desktop AppShell

shadcn/ui 매핑:

- `Sheet`: 모바일 사이드바
- `Button`: 메뉴/액션
- `Badge`: 상태/카운트
- `Separator`: 네비게이션 구분
- `Tooltip`: 아이콘 액션 설명

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ Sidebar                  │ Top bar                                           │
│                          │ Brand: J Travel        Last generated: 10:00      │
│ Brand Pilot              │ Channels: IG connected · Webflow mapping required │
│                          ├───────────────────────────────────────────────────┤
│ 온보딩                   │                                                   │
│ 콘텐츠        [3]        │ Page header                                       │
│ 게시 큐       [1]        │ Title / Description / Primary action              │
│ 소스                     │                                                   │
│ 채널          [!]        │ Page content                                      │
│ 브랜드 설정              │                                                   │
│                          │                                                   │
│                          │                                                   │
└──────────────────────────────────────────────────────────────────────────────┘
```

규칙:

- 대시보드 메뉴는 없다.
- 사이드바 폭은 desktop 기준 240px.
- 본문은 `max-w-screen-2xl` 안에서 운영툴 밀도를 유지한다.
- 상단 바는 브랜드, 연결 상태 요약, 마지막 생성 시간을 표시한다.
- 실패/검토 필요/권한 오류는 메뉴 배지로 표시한다.

## 3. 온보딩

### 3.1 목적

고객이 콘텐츠 생성과 자동 게시를 시작하기 위한 필수 조건을 단계별로 완료한다.

### 3.2 Desktop Wireframe

shadcn/ui 매핑:

- `Card`: 체크리스트 컨테이너
- `Badge`: 단계 상태
- `Button`: 단계 이동
- `Alert`: 오류/확인 필요
- `Progress`: 완료율

```text
┌──────────────────────────────────────────────────────────────────────┐
│ 온보딩                                                               │
│ 콘텐츠 자동 운영을 시작하기 위한 필수 설정을 완료하세요.             │
│                                                    [첫 콘텐츠 생성]  │
├──────────────────────────────────────────────────────────────────────┤
│ Progress  ███████░░░  70%                                            │
├──────────────────────────────────────────────────────────────────────┤
│ [완료] 브랜드 정보 입력              J Travel             [보기]      │
│ [완료] 자사 URL 등록                 5개 등록             [관리]      │
│ [필요] 참고 URL 등록                 선택 사항            [추가]      │
│ [완료] 주제표 업로드                 128개 유효           [보기]      │
│ [완료] Instagram 연결                @jtravel            [확인]      │
│ [필요] Threads 연결                  연결 전              [연결]      │
│ [오류] Webflow 연결                  필드 매핑 필요       [수정]      │
│ [필요] 첫 콘텐츠 생성                아직 생성 전         [생성]      │
│ [필요] 첫 승인                       검토 항목 없음       [대기]      │
│ [필요] 첫 게시 완료                  게시 이력 없음       [대기]      │
└──────────────────────────────────────────────────────────────────────┘

Alert:
┌──────────────────────────────────────────────────────────────────────┐
│ Webflow 필드 매핑이 필요합니다.                                      │
│ Webflow 게시를 시작하려면 title, slug, body 필드를 매핑하세요. [수정] │
└──────────────────────────────────────────────────────────────────────┘
```

### 3.3 Empty / Error

빈 상태:

```text
브랜드 설정이 아직 없습니다.
콘텐츠를 생성하려면 브랜드 정보가 먼저 필요합니다.
[브랜드 설정하기]
```

오류 상태:

```text
시작 조건을 확인할 수 없습니다.
일시적인 오류일 수 있습니다.
[다시 시도]
```

### 3.4 Responsive

- Desktop: 체크리스트 행 기반.
- Mobile: 단계별 카드 리스트.
- 오류 alert는 체크리스트 위에 고정한다.

## 4. 콘텐츠

### 4.1 목적

- `Tabs`: 검토 상태 탭
- `Table`: 콘텐츠 리스트
- `Card`: 상세 패널
- `Badge`: 상태
- `Button`: 승인/재생성/거절
- `Dialog`: 재생성 사유 입력
- `Alert`: 자동 승인 차단 사유

- Mobile: 리스트에서 항목 선택 시 상세 화면으로 이동.

## 7. 채널

Instagram, Threads, Webflow 연결 상태와 게시 권한을 관리한다.

### 7.2 Desktop Wireframe

shadcn/ui 매핑:

- `Card`: 채널 카드
- `Badge`: 연결 상태
- `Button`: 연결/확인/재연결
- `Alert`: 권한/토큰/매핑 오류
- `Dialog`: 연결 정보 입력, Webflow mapping

```text
┌──────────────────────────────────────────────────────────────────────────────┐
```

## 11. Responsive Rules

Global:

- `lg` 이상: sidebar 고정, 본문 2-column/split view 허용.
- `md` 이상: 테이블 유지 가능.
- `md` 미만: 테이블을 카드 리스트로 전환.
- `lg` 미만: sidebar를 sheet로 전환.

Screen-specific:

- 콘텐츠: desktop은 split view, mobile은 list-to-detail navigation.
- 게시 큐: desktop은 슬롯 테이블, mobile은 슬롯 카드.
- 채널: desktop은 3-column card grid, mobile은 single-column.
- 브랜드 설정: desktop은 tab form, mobile은 accordion 또는 stacked tab.

## 12. 구현 전 체크리스트

- [ ] AppShell에 대시보드 메뉴가 없다.
- [ ] 모든 상태 배지는 텍스트를 포함한다.
- [ ] 검토 화면에는 직접 편집 버튼이 없다.
- [ ] 게시 큐에는 시간 수정 버튼이 없다.
- [ ] 자동 승인 차단 사유가 콘텐츠 상세에서 보인다.
- [ ] Webflow 필드 매핑 오류가 채널 화면으로 연결된다.
- [ ] 모바일에서 sidebar는 Sheet로 동작한다.
- [ ] 테이블은 모바일에서 카드 리스트로 전환된다.
- [ ] 오류 메시지는 원인과 다음 액션을 포함한다.
- [ ] 아이콘 버튼에는 tooltip 또는 sr-only label이 있다.

## 13. 자체 검토

- 고객용 IA 6개 화면을 모두 포함했다.
- 대시보드를 추가하지 않았다.
- 운영자 화면을 추가하지 않았다.
- 직접 편집, 게시 시간 수정, 채널별 개수 설정을 추가하지 않았다.
- React, Next.js, Tailwind, shadcn/ui로 구현 가능한 컴포넌트 매핑을 포함했다.
- 접근성 기준인 focus, label, dialog, mobile touch target을 반영했다.
