# Brand Pilot Cognitive Load / Conversion Review

작성일: 2026-07-05  
대상: 고객용 HTML 프로토타입 6개 화면  
기준: `cognitive-load-conversion` skill

## 1. Extraneous Load Found

### 온보딩 체크리스트가 너무 길었다

기존 온보딩은 완료된 항목과 막힌 항목이 같은 무게로 노출돼 사용자가 "지금 뭘 해야 하는지" 바로 파악하기 어려웠다.

수정:

- 막힌 항목 3개를 먼저 보여주도록 헤드라인과 체크리스트를 정리했다.
- 완료된 항목은 별도 요약 카드로 낮은 우선순위에 배치했다.

### 하위 상태를 독립 페이지로 과도하게 분리했다

자동 승인, 게시 실패, URL 추가, 채널 점검 같은 하위 상태가 별도 HTML로 분리되어 탐색 비용이 커졌다.

수정:

- 고객용 주요 화면은 6개만 유지했다.
- 하위 상태는 상위 화면의 탭/섹션으로 통합했다.

### 소스/채널 화면의 보조 폼이 항상 노출됐다

URL 추가, 크롤링 결과, 주제표 검증 결과, 채널 점검 결과가 화면 하단에 항상 떠 있어 기본 업무를 방해했다.

수정:

- 보조 섹션은 `details`로 접어서 필요할 때만 열리게 했다.

## 2. Mental Model Gaps

### 탭처럼 보이지만 페이지 이동하는 링크가 있었다

사용자는 탭을 같은 화면 내 전환으로 기대한다. 이전 구조는 일부 탭이 별도 HTML로 이동했다.

수정:

- 콘텐츠, 게시 큐, 소스, 채널의 탭은 같은 페이지 안에서 전환되도록 정리했다.

### 버튼과 비활성 상태가 섞여 있었다

조건 미충족 상태도 버튼처럼 보여 클릭 가능해 보였다.

수정:

- 비활성 항목은 `button`이 아니라 `aria-disabled` 표시 요소로 바꿨다.

## 3. Offloading Opportunities

### 사용자가 다음 할 일을 해석하지 않게 해야 한다

온보딩에서 "6/9 완료"만으로는 다음 행동을 추론해야 한다.

권장:

- 현재 막힌 항목을 최상단에 요약한다.
- "지금 해야 할 일" CTA를 1개만 강하게 보여준다.

### 채널 연결 오류는 기술 용어보다 복구 행동을 먼저 보여줘야 한다

권한, 토큰, 계정 타입 같은 용어는 필요한 정보지만 첫 문장으로는 무겁다.

권장:

- "Instagram 게시 테스트가 아직 안 됐습니다"처럼 사용자 행동 중심으로 표기한다.
- 상세 기술 사유는 펼침 영역에 둔다.

### 자동 승인 차단 사유는 콘텐츠 카드에서 바로 보여야 한다

사용자가 차단 탭으로 들어가기 전에도 "왜 검토가 필요한지"를 알 수 있어야 한다.

권장:

- 검토 목록 카드에 차단/검토 사유 1줄을 표시한다.

## 4. Priority Recommendations

### 1. 온보딩의 첫 CTA를 하나로 줄이기

변경:

- 현재는 `브랜드 설정`, `URL 추가`가 동시에 보인다.
- 다음 단계 기준으로 `채널 연결 확인` 하나를 primary로 두는 편이 낫다.

효과: High  
제거되는 비용: 사용자가 둘 중 무엇을 먼저 할지 판단하는 비용

### 2. 콘텐츠 검토 화면의 액션 우선순위 정리

변경:

- `승인`은 primary, `재생성`은 secondary, `거절`은 가장 약하게 둔다.
- 지금은 세 액션이 같은 영역에서 비슷한 무게로 보인다.

효과: High  
제거되는 비용: 승인/재생성/거절 사이의 결정 피로

### 3. 소스 화면의 URL 추가 폼은 기본 접힘 유지

변경:

- 기본은 자사 URL 목록과 주제표 상태 확인.
- 추가/검증 결과는 요청 시 펼침.

효과: Medium  
제거되는 비용: 관리 화면에서 보조 폼을 계속 읽는 비용

### 4. 채널 화면의 오류 메시지를 복구 행동 중심으로 바꾸기

변경:

- "토큰 필요"보다 "Threads 게시를 시작하려면 연결 정보가 필요합니다"처럼 작성.

효과: Medium  
제거되는 비용: 기술 오류를 사용자가 해석하는 비용

### 5. 게시 큐에서 실패를 현재 탭 안에서 직접 해결하게 하기

변경:

- 실패 탭에서 바로 `채널 상태 확인`, `이미지 URL 확인` 액션을 제공한다.

효과: Medium  
제거되는 비용: 실패 원인과 해결 화면을 따로 찾아야 하는 비용

## 5. Applied Changes

- 온보딩 헤드라인을 "남은 3개 해결" 중심으로 변경
- 완료 항목을 별도 요약 카드로 이동
- 하위 HTML 페이지 24개 제거 후 6개 화면으로 정리
- 탭을 같은 페이지 내 전환으로 정리
- 소스/채널의 보조 상태 섹션을 접힘 영역으로 변경
- 버튼 없는 비활성 상태를 `aria-disabled` 표시 요소로 변경

## 6. Remaining Risk

- 현재 HTML 프로토타입은 정적 상태라 사용자가 탭 클릭 후 스크롤 위치 이동까지 완전히 자연스럽지는 않다.
- React 구현 시 탭 상태, URL hash, focus 이동을 실제 컴포넌트에서 처리해야 한다.
- 온보딩 CTA는 실제 채널 연결 정책이 확정되면 한 번 더 정리해야 한다.
