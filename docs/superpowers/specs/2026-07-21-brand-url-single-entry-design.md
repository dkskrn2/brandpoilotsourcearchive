# 브랜드 URL 단일 입력 설계

## 목적

브랜드 설정에서 자사 URL을 두 곳에서 관리하는 것처럼 보이는 문제를 제거한다. 사용자는 브랜드 정보 분석 흐름에서만 대표 URL을 입력하고, 확정된 URL은 크롤링, Wiki, DM 답변, 콘텐츠 생성이 공통으로 사용한다.

## 확정 동작

- 브랜드 설정의 별도 `자사 URL` 관리 패널은 표시하지 않는다.
- `확정된 브랜드 정보`의 `브랜드 정보 만들기` 또는 `브랜드 정보 다시 분석` 흐름에서 대표 URL을 입력한다.
- 분석 요청에 입력한 URL은 분석 실행의 근거 스냅샷으로 저장한다.
- 분석 시작, 실패, 사용자 이탈 시 현재 운영 중인 자사 URL은 변경하지 않는다.
- 사용자가 분석 결과에서 `확인하고 저장`을 누를 때만 대표 URL을 `source_urls`의 자사 URL과 동기화한다.
- 과거 분석 실행의 입력 URL은 감사와 재현을 위해 변경하지 않는다.

## 데이터 흐름

1. 브랜드 정보 분석 화면은 현재 활성 자사 URL을 초기값으로 조회한다.
2. 사용자가 URL과 선택 문서를 제출하면 `brand_analysis_runs.input_json`에 입력 스냅샷을 저장한다.
3. CLI 워커는 해당 스냅샷을 근거로 분석한다.
4. 사용자가 결과를 수정한 뒤 확정한다.
5. 확정 트랜잭션에서 다음을 함께 처리한다.
   - 분석 실행을 활성 확정 버전으로 지정
   - 호환 브랜드 프로필 필드 갱신
   - 입력 URL이 있으면 브랜드의 자사 URL을 생성 또는 갱신
   - Wiki 재생성 요청
6. URL이 변경되면 새 URL은 이후 크롤링과 모든 콘텐츠·DM 근거 조회에 사용한다.

## URL 동기화 규칙

- 입력 URL이 현재 자사 URL과 같으면 `source_urls`를 변경하지 않는다.
- 기존 자사 URL이 없으면 활성 자사 URL을 한 건 생성한다.
- 기존 자사 URL과 다르면 같은 행의 URL, URL 해시, 도메인, 크롤링 상태를 갱신한다.
- URL을 문서 없이 비워서 분석한 경우 기존 자사 URL을 삭제하거나 비활성화하지 않는다.
- URL 변경 후 초기 크롤링 실패는 브랜드 정보 확정을 되돌리지 않는다. 크롤링 오류를 자사 URL 상태에 기록하고 재시도할 수 있게 한다.

## UI

- 브랜드 설정에서 `OwnedSourceSettings` 패널을 제거한다.
- 확정된 브랜드 정보 영역은 현재 대표 URL과 마지막 분석 일시를 읽기 전용으로 표시한다.
- URL 수정은 `브랜드 정보 다시 분석` 버튼으로 진입한다.
- 분석 화면의 URL 입력은 현재 대표 URL로 미리 채운다.
- 확정 전에는 기존 운영 URL이 유지된다는 안내를 분석 화면에 표시한다.

## 오류 처리

- URL 형식과 안전성 검증은 기존 자사 URL·크롤러 검증을 재사용한다.
- 다른 활성 자사 URL 행을 추가하지 않고 기존 한 행을 갱신해 브랜드당 한 개 제약을 유지한다.
- URL 중복이나 충돌이 발생하면 전체 확정 트랜잭션을 롤백하고 사용자에게 저장 실패를 알린다.
- 크롤링은 확정 트랜잭션 밖에서 수행해 외부 네트워크 지연이 저장을 막지 않게 한다.

## 검증

- 기존 URL이 없는 브랜드의 분석 확정 시 자사 URL 생성
- 기존 URL과 같은 URL을 확정할 때 중복 생성 없음
- 다른 URL을 확정할 때 기존 자사 URL 행 갱신
- 분석 실패와 확정 전 이탈 시 기존 URL 유지
- 문서만으로 분석 확정 시 기존 URL 유지
- URL 변경 확정 후 Wiki 요청 생성
- 브랜드 설정에서 별도 URL 편집 패널이 보이지 않음
- 다시 분석 화면에 현재 URL이 초기값으로 표시됨

## 제외 범위

- 과거 분석 실행 URL 변경
- 여러 자사 URL 지원
- 확정 전에 운영 URL 변경
- URL 변경과 외부 크롤링을 하나의 장시간 트랜잭션으로 처리
