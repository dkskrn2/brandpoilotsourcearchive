# 고객 런타임 안정성 및 설정 정보 구조 개선 설계

## 목표

AI 콘텐츠 결과 미리보기와 Instagram 게시, DM 수동 답변의 실패 원인을 추적 가능하게 만들고, 아카이브·최근 검색·브랜드 분석 강제 진입·자동승인 설정을 사용자의 기대와 일치시킨다.

## 확인된 현황

- 생성 `3fa56155-de02-4b85-81bd-c8b5053593e9`와 카드뉴스 결과는 정상 완료됐고 서로 다른 이미지 3장이 저장되어 있다.
- 같은 결과의 Instagram Story는 게시 ID `18130501201627059`를 받아 성공했다.
- Instagram 캐러셀은 준비 시작 약 1.5초 뒤 `instagram_publish_failed`로 끝났다. 현재 저장 로직이 원래 예외를 일반 코드로 축약해 실패 단계와 Meta 응답 코드를 복원할 수 없다.
- 캐러셀 미리보기의 index와 숫자는 바뀌지만 한 이미지 요소의 `src` 교체 방식에서는 사용자 화면의 이미지가 바뀌지 않는다.
- 현재 Instagram DM 연결은 활성 `instagram_login` 자격 증명을 갖고 있지만 수동 발송 감사 레코드는 없다. 과거 실패가 사전검증에서 종료됐거나 당시 실행 코드가 현재 코드와 달랐을 수 있다.
- 브랜드 설정 게이트는 활성 브랜드 분석이 없으면 `/onboarding/brand-intelligence`로 이동시킨다.
- 아카이브에는 저장 API만 연결되어 있고 저장 해제 동작이 없다.
- 최근 검색어 삭제는 원격 DELETE 응답 뒤에 로컬 목록을 갱신해 네트워크 지연이 그대로 노출된다.
- Instagram 콘텐츠 형식 설정은 자동승인 영역과 분리된 별도 아코디언이다.

## 설계

### 1. 결과 미리보기 캐러셀

모든 자산을 순서대로 가로 트랙에 렌더하고 현재 index에 따라 트랙을 `translateX(-index * 100%)` 이동한다. 숫자, 점 표시, 이전·다음 버튼은 동일한 index를 사용한다. 자산 목록이 바뀌면 index를 0으로 초기화하고 현재 index가 범위를 벗어나지 않게 제한한다. 각 슬라이드는 자산 URL과 index를 결합한 안정적인 key를 사용한다.

### 2. Instagram 게시 오류 관측성

게시 흐름을 다음 단계로 구분한다.

1. manifest 읽기 및 검증
2. 캐러셀 자식 미디어 컨테이너 생성·상태 확인
3. 부모 캐러셀 컨테이너 생성·상태 확인
4. `media_publish`

Meta Graph 오류에는 HTTP 상태, Meta code, subcode와 실패 단계를 포함하되 access token, 원문 응답, 사용자 데이터는 저장하지 않는다. `publish_attempts.response_metadata`에는 `stage`, `httpStatus`, `metaCode`, `metaSubcode`, `retryable`을 기록한다. UI는 안정적인 내부 오류 코드로 사용자가 취할 조치를 안내한다. 기존 실패 건은 원본 오류가 소실됐으므로 수정 후 사용자가 명시적으로 재시도할 때 새 진단 정보를 수집한다. 자동 재게시로 외부 중복 게시를 만들지 않는다.

### 3. DM 수동 답변 오류 관측성

수동 발송 요청은 사전검증부터 감사 레코드를 남긴다. 실패 단계를 `conversation`, `channel`, `credential`, `recipient`, `meta_send`, `message_persist`로 구분한다. Meta 요청 실패는 Instagram 게시와 동일한 비민감 오류 메타데이터 규칙을 따른다. 멱등 키를 유지해 같은 요청의 중복 전송을 방지한다. 전송 여부가 불명확한 경우 자동 재전송하지 않고 `unknown`으로 남긴다.

### 4. 브랜드 분석 강제 진입

완료 조건은 `active_brand_analysis_id` 존재로 유지한다. 미완료 사용자는 지원 페이지와 브랜드 분석 페이지를 제외한 모든 보호 경로에서 `/onboarding/brand-intelligence`로 이동한다. 기존 프로필 텍스트만 있고 활성 분석이 없는 계정도 분석 대상으로 본다.

### 5. 아카이브 북마크 토글

백엔드에 브랜드 범위의 저장 해제 API를 추가한다. 저장된 카드와 상세 대화상자의 북마크 버튼을 활성화하고 클릭 즉시 로컬 상태에서 체크를 해제한다. 아카이브 화면에서는 즉시 목록에서 제거하고 total을 감소시킨다. 요청 실패 시 카드와 total을 원상복구하고 오류를 표시한다. 저장/해제 요청이 진행 중인 항목만 비활성화한다.

### 6. 최근 검색어 삭제

삭제 클릭 즉시 해당 검색어를 로컬 목록에서 제거한다. DELETE는 백그라운드에서 실행하고 실패 시 원래 위치에 복구한 뒤 오류를 표시한다. 동시에 여러 항목을 삭제할 수 있도록 단일 전역 삭제 상태 대신 항목 ID 집합을 사용한다. 휴지통 아이콘은 X 아이콘으로 바꾸고 접근성 이름은 `최근 검색 #태그 삭제`를 유지한다.

### 7. 자동승인과 채널별 콘텐츠 형식

설정 구조를 다음처럼 구성한다.

- 자동 승인
  - 브랜드 전체 자동 승인 스위치
  - Instagram
    - 카드뉴스
    - 릴스
    - 스토리

채널 블록은 재사용 가능한 구조로 만들되 현재는 Instagram만 렌더한다. 이후 Threads, TikTok 등은 같은 자동승인 영역에 채널 블록을 추가한다. 기존 API의 Instagram 형식 계약과 저장 순서는 유지한다.

## 오류 처리 및 안전성

- 외부 게시와 DM은 사용자의 명시적 동작 없이 재시도하지 않는다.
- 낙관적 UI 변경은 서버 실패 시 반드시 복구한다.
- 자격 증명, 토큰, Meta 원문 응답은 로그와 DB에 기록하지 않는다.
- 게시 성공 후 내부 저장만 실패한 경우 이미 외부 게시된 상태를 우선 보존하고 중복 발행을 막는다.

## 테스트

- 캐러셀 버튼 클릭 시 index, 숫자, 점, 트랙 transform이 함께 변경되는 컴포넌트 테스트
- 서로 다른 세 자산에서 실제 활성 슬라이드가 바뀌는 브라우저 회귀 테스트
- 캐러셀 자식·부모·발행 각 단계의 Meta 오류 분류와 비민감 메타데이터 저장 테스트
- DM 사전검증 및 Meta 실패 단계별 감사 레코드, 멱등성, unknown 상태 테스트
- 브랜드 분석 미완료/완료/허용 경로별 게이트 테스트
- 아카이브 저장 해제 성공과 실패 롤백 테스트
- 최근 검색어 즉시 제거, 병렬 삭제, 실패 복구, X 버튼 접근성 테스트
- 자동승인 내부 Instagram 형식 렌더와 저장 회귀 테스트

## 범위 제외

- 기존 실패 캐러셀을 자동으로 다시 게시하지 않는다.
- Threads·TikTok 콘텐츠 형식 자체는 이번 변경에서 구현하지 않는다.
- 전체 게시 시스템을 새로운 공통 상태 머신으로 교체하지 않는다.
