# AI 콘텐츠 헤더 사용량 표시 설계

## 목적

AI 콘텐츠 생성 화면에서 별도 사용량 영역을 읽지 않아도 오늘 남은 생성 및 다운로드 횟수를 바로 확인할 수 있게 한다.

## 화면 구성

- `AI 콘텐츠 생성` 페이지 헤더 오른쪽에 `생성 N회 남음`, `다운로드 N회 남음`을 표시한다.
- 사용량 표시 다음에 기존 `새 콘텐츠 만들기` 버튼을 유지한다.
- 기존의 큰 `오늘 사용량` 섹션은 제거한다.
- 잔여량은 `max(한도 - 사용량, 0)`으로 계산한다.
- 모바일에서는 사용량과 버튼이 자연스럽게 줄바꿈되며 텍스트가 잘리지 않아야 한다.

## 데이터 흐름

- 기존 `GET /brands/:brandId/ai-content/usage` 응답과 `AiContentUsage` 상태를 그대로 사용한다.
- 추가 API 요청이나 DB 변경은 하지 않는다.
- 페이지 로딩 및 오류 처리는 현재 동작을 유지한다.

## 검증

- 생성·다운로드 잔여량이 헤더에 계산되어 표시되는지 컴포넌트 테스트로 확인한다.
- 사용량이 한도를 초과하더라도 음수가 표시되지 않는지 확인한다.
- 기존 `오늘 사용량` 섹션이 제거됐는지 확인한다.
- 고객 UI 테스트와 빌드를 실행한다.
