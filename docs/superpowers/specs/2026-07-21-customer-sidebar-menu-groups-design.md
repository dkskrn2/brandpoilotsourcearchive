# 고객 사이드바 메뉴 그룹 설계

작성일: 2026-07-21

상태: 사용자 승인

## 목적

고객 사이드바의 기존 기능과 경로는 유지하면서 메뉴를 사용 목적별로 묶어 탐색성을 높인다. 별도 관리자 콘솔로 이전된 임시 고객 UI 관리자 화면은 제거한다.

## 메뉴 구성

| 그룹 | 메뉴 |
| --- | --- |
| 개요 | 대시보드 |
| 콘텐츠 운영 | AI 콘텐츠 생성, 소스, 트렌드 탐색, 게시 관리 |
| 채널·고객 | 채널, DM 자동답변 |
| 설정·지원 | 브랜드 설정, 결제 및 구독, 고객센터 |
| 시작 준비 | 온보딩. 미완료 항목이 있을 때만 표시 |

## 동작 보존

- 기존 URL, 활성 메뉴 표시, 배지 수치, 브랜드 설정 완료 전 잠금 조건을 변경하지 않는다.
- 결제 및 구독은 기존 외부 가격 페이지 링크를 유지한다.
- 온보딩 그룹은 온보딩 메뉴가 숨겨지면 함께 렌더링하지 않는다.
- 모바일에서도 그룹 제목과 메뉴 순서를 유지한다.

## 관리자 화면 정리

- 고객 사이드바의 `관리자 채널` 메뉴를 제거한다.
- 고객 UI의 `/admin/channels` 라우트, 페이지, 전용 테스트와 도움말 항목을 제거한다.
- 실제 관리 기능은 `dkskrn2/main`의 `/admin/brand-pilot/*`와 중앙 API `/admin/v1/*`를 사용한다.
- 고객센터 설명에서 임시 관리자 채널을 가리키는 문구를 일반적인 관리자 답변 안내로 바꾼다.

## 검증

- 사이드바 그룹 제목과 그룹별 메뉴 순서를 컴포넌트 테스트로 검증한다.
- `관리자 채널` 메뉴와 `/admin/channels` 라우트가 제거되었는지 검증한다.
- 기존 배지, 잠금, 외부 결제 링크 테스트를 유지한다.
- 고객 UI 테스트와 프로덕션 빌드를 실행한다.
