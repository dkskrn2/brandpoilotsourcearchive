# 게시 결과 상세 미리보기 및 저장 설계

작성일: 2026-07-15  
상태: 확정안 검토  
대상: 고객용 게시 관리 화면

## 1. 목적

게시 관리에서 완료되거나 실패한 채널 결과를 열었을 때 내부 JSON 대신 실제 업로드 콘텐츠를 확인한다. 사용자는 같은 화면에서 게시 정보와 생성 근거를 확인하고, 현재 개발 단계에서는 결과물 전체를 ZIP으로 저장할 수 있다.

지원 대상은 카드뉴스, 단일 이미지, 스토리, 영상, 릴스, HTML, 텍스트다. 알 수 없는 형식도 빈 화면으로 끝내지 않고 원본 링크와 오류 안내를 제공한다.

## 2. 화면 구조

선택안은 미디어 중심 분할형이다.

1. 고정 헤더
   - 콘텐츠 제목
   - 채널과 콘텐츠 형식
   - 게시 상태
   - 닫기 버튼
2. 스크롤 본문
   - 왼쪽 65%: 실제 콘텐츠 미리보기
   - 오른쪽 35%: 업로드 정보
3. 고정 하단
   - 저장 버튼
   - 원본 게시 URL이 있을 때만 원본 게시물 열기 버튼

모달은 화면 높이를 넘지 않는다. 헤더와 하단 액션은 고정하고 본문만 스크롤한다. 좁은 화면에서는 미리보기와 업로드 정보를 한 열로 쌓는다.

## 3. 콘텐츠 형식별 미리보기

### 3.1 카드뉴스

- 선택된 이미지를 큰 정방형 영역에 표시한다.
- 하단에 전체 이미지 썸네일을 순서대로 표시한다.
- 썸네일 선택 시 큰 이미지가 바뀐다.
- 이미지 개수는 실제 manifest를 기준으로 하며 고정하지 않는다.
- 이미지는 잘라내지 않고 `contain`으로 표시한다.

### 3.2 단일 이미지와 스토리

- 원본 가로세로 비율을 유지한다.
- 세로 이미지가 화면보다 길면 미리보기 영역 안에서 스크롤하거나 축소한다.

### 3.3 영상과 릴스

- 브라우저 기본 영상 플레이어를 사용한다.
- 자동 재생하지 않는다.
- 포스터, 재생 시간, 재생 컨트롤을 표시한다.
- `controlsList="nodownload"`와 Picture-in-Picture 비활성화를 적용한다.

### 3.4 HTML

- 스크립트를 실행하지 않는 격리된 미리보기로 표시한다.
- HTML이 길면 미리보기 내부에서 스크롤한다.
- 렌더링할 수 없으면 HTML 파일명과 원본 열기 안내를 표시한다.

### 3.5 텍스트 및 알 수 없는 형식

- 텍스트는 실제 게시 본문을 읽기 좋은 폭으로 표시한다.
- 형식 판별에 실패하면 이용 가능한 제목, 본문, 파일 링크를 표시한다.
- 저장된 JSON 전체는 고객 화면에 노출하지 않는다.

## 4. 업로드 정보

오른쪽 정보 영역에는 다음 값만 표시한다.

- 채널
- 콘텐츠 형식
- 게시 상태
- 게시 시각 또는 실패 시각
- 외부 게시 ID
- 외부 게시 URL
- 생성 근거
- 실패한 경우 오류 사유

값이 없는 항목은 불필요하게 나열하지 않는다. 외부 게시 ID는 긴 전체 문자열 대신 식별 가능한 마지막 값만 표시한다.

## 5. API 및 데이터 계약

프론트가 채널별 manifest 구조를 직접 해석하지 않도록 중앙 API가 정규화한다.

### 5.1 결과물 조회

`GET /publish-queue/:queueId/artifacts`

응답 예시:

```json
{
  "queueId": "queue-id",
  "kind": "image_gallery",
  "deliveryFormat": "instagram_feed_carousel",
  "assets": [
    {
      "url": "https://storage.example/card-01.png",
      "fileName": "card-01.png",
      "mimeType": "image/png",
      "width": 1080,
      "height": 1080
    }
  ],
  "posterUrl": null,
  "html": null,
  "text": null
}
```

`kind`는 `image_gallery`, `image`, `video`, `html`, `text`, `unknown` 중 하나다. API는 `storage_artifacts.public_url`의 manifest와 `channel_outputs.output_json`을 읽어 이 구조로 변환한다.

### 5.2 개별 저장

`GET /publish-queue/:queueId/download`

- 해당 큐의 결과만 ZIP으로 반환한다.
- 카드뉴스는 모든 이미지와 manifest를 포함한다.
- 영상은 영상, 포스터, manifest를 포함한다.
- HTML은 HTML 파일과 연결된 자산을 포함한다.
- 공통으로 게시 정보, 게시 URL, 생성 근거를 포함한다.
- 파일이 일부 누락되어도 나머지 파일은 저장하고 누락 목록을 ZIP에 기록한다.

기존 브랜드 전체 완료 결과 다운로드 API는 유지한다.

## 6. 저장 권한 정책

### 현재 개발 단계

- 저장 버튼을 활성화한다.
- 내부 기능 검증을 위해 별도 결제 권한을 요구하지 않는다.

### 결제 연동 이후

- 동일한 다운로드 API에서 서버가 구독 상태와 저장 권한을 확인한다.
- 권한이 없으면 `403 download_entitlement_required`를 반환한다.
- 프론트는 결제 페이지 이동 안내를 표시한다.
- 원본 ZIP URL을 프론트에 직접 전달하지 않는다.

## 7. 브라우저 저장 억제와 한계

현재 미리보기에는 다음 억제 수단을 적용한다.

- 미리보기 영역의 우클릭 메뉴 차단
- 이미지 드래그 차단
- 영상 다운로드 메뉴와 Picture-in-Picture 비활성화
- 원본 URL을 화면 텍스트로 표시하지 않음

이 조치는 사용 편의상 저장 동작을 제한하는 것이며 보안 경계가 아니다. 브라우저에 원본 데이터가 전달되면 개발자 도구, 네트워크 기록, 화면 캡처로 복제할 수 있다.

결제 권한을 실제로 강제할 때는 다음 구조로 전환한다.

- 브라우저에는 저해상도 또는 워터마크가 적용된 미리보기 파생본만 전달
- 원본 저장은 인증된 다운로드 API로만 제공
- 원본 스토리지는 비공개로 전환하고 짧은 수명의 서버 서명 URL 또는 서버 스트리밍 사용

## 8. 상태 및 오류 처리

- 로딩 중: 미리보기와 정보 영역에 로딩 상태 표시
- manifest 오류: 기본 제목과 본문을 표시하고 재시도 제공
- 일부 파일 누락: 이용 가능한 파일만 표시하고 누락 개수 안내
- 영상 재생 실패: 포스터와 저장 버튼 유지
- HTML 렌더링 실패: 파일 안내와 저장 버튼 유지
- 저장 실패: 모달을 닫지 않고 오류 안내
- 저장 권한 없음: 결제 페이지 이동 안내

## 9. 테스트 범위

### 프론트

- 형식별 미리보기 분기
- 카드뉴스 썸네일 전환
- 긴 콘텐츠와 긴 업로드 정보의 스크롤
- 우클릭 및 드래그 억제
- 저장 성공, 실패, 권한 없음 상태
- 모바일 단일 열 배치

### API

- manifest 형식별 정규화
- 개별 큐만 ZIP에 포함되는지 확인
- 여러 이미지와 영상 자산 포함
- 파일 일부 누락 시 나머지 결과 유지
- 다른 브랜드 큐 접근 차단
- 향후 다운로드 권한 검사 지점 유지

## 10. 제외 범위

- 미리보기에서 콘텐츠 편집
- 게시물 재업로드
- 결제 또는 토스페이먼츠 실제 연동
- 완전한 원본 보호용 비공개 스토리지 전환
- 미디어 변환 및 워터마크 생성
