# AI 콘텐츠 워커 제작 품질 가이드

작성일: 2026-07-18  
상태: 워커 Skill 반영 기준  
적용 대상: 카드뉴스, 블로그, 마케팅 소재 전용 Codex CLI 워커

## 1. 목적

인터넷에서 확인한 공식 플랫폼 가이드와 웹 가독성 연구를 워커별 실행 규칙으로 정리한다. 워커는 생성할 때 인터넷의 문장을 복사하지 않는다. 사용자가 제공한 브랜드 정보, 자사 URL, 제품 URL과 선택한 레퍼런스를 근거로 새 콘텐츠를 만들고 이 문서의 제작 원칙으로 구성과 품질을 점검한다.

이 문서는 프롬프트에 통째로 붙이는 참고 글이 아니다. 각 워커의 `SKILL.md`에 필요한 규칙만 옮기고, 산출물 schema와 금지 조건은 코드 parser에서도 확인한다.

## 2. 공통 제작 원칙

세 워커는 다음 순서로 작업한다.

1. **근거 추출**
   - 자사·제품 URL과 사용자가 입력한 정보에서 확인 가능한 사실, 혜택, 조건과 금지 표현을 분리한다.
   - 확인되지 않은 가격, 수치, 후기, 성과와 기간 한정 표현을 만들지 않는다.
2. **사용자 가치 정의**
   - 선택된 타깃이 어떤 상황에서 이 콘텐츠를 보고 무엇을 이해하거나 행동해야 하는지 한 문장으로 정한다.
   - 단순 요약보다 문제 해결, 의사결정, 공감, 저장 또는 공유 가치 중 하나를 우선한다.
3. **구성 기획**
   - 제목, 정보 순서, 핵심 메시지와 CTA를 먼저 만든다.
   - 레퍼런스는 정보 구조와 표현 방식만 참고하고 문구나 디자인을 복제하지 않는다.
4. **최종 제작**
   - 브랜드의 주색, 톤, 금지 표현과 사용자가 지정한 비율을 반영한다.
   - 생성 결과에 근거 URL 문자열을 노출하지 않는다.
5. **자체 점검**
   - 사실성, 구체성, 중복, 가독성, 형식 일치와 CTA 적합성을 확인한다.
   - 검사 실패 부분만 한 번 수정하고 전체 결과를 무조건 다시 만들지 않는다.

공통 금지 항목은 다음과 같다.

- 근거 없는 최고·유일·보장 표현
- 충격, 공포와 과장을 이용한 클릭 유도
- 같은 뜻의 문장을 제목, 본문과 CTA에서 반복
- 실제로 누를 수 없는 재생·다운로드·자세히 보기 버튼을 이미지에 그림
- 레퍼런스의 문장, 캐릭터, 로고와 레이아웃을 그대로 복제
- 정보를 채우기 위한 임의의 숫자, 통계, 후기와 할인 조건 생성

### 2.1 자연스러운 한국어와 실제 경험 사용

목표는 AI 탐지기를 피하는 것이 아니라 독자가 읽었을 때 구체적이고 자연스러우며 브랜드가 직접 설명한 것처럼 느끼는 콘텐츠다.

- 자사 사이트, 제품 설명, FAQ와 기존 성과 우수 콘텐츠에서 브랜드가 실제로 사용하는 단어를 우선한다.
- 추상적인 장점보다 사용 상황, 선택 기준, 실수하기 쉬운 지점과 구체적인 다음 행동을 쓴다.
- 모든 문장을 같은 길이와 문형으로 정렬하지 않는다. 짧은 핵심 문장과 설명 문장을 자연스럽게 섞는다.
- 번역투, 불필요한 존칭, 과한 감탄과 홍보 문구를 줄인다.
- 도입, 각 절의 끝과 결론에서 같은 내용을 반복하지 않는다.
- `오늘날`, `빠르게 변화하는`, `단순한 A를 넘어 B`, `핵심은`, `결론적으로`, `혁신적인`, `완벽한 솔루션`처럼 근거 없이 자주 반복되는 상투 표현을 기본값으로 사용하지 않는다.
- `첫째`, `둘째`, `셋째`를 기계적으로 붙이지 않는다. 실제 순서가 중요한 절차에만 번호를 사용한다.
- 사용자가 제공한 경험, 후기, 사례가 있으면 주체와 조건을 유지해 활용한다.
- 실제 경험 근거가 없으면 `직접 사용해 보니`, `고객들이 공통으로 말합니다`, `현장에서 자주 봤습니다` 같은 1인칭 경험과 집단 반응을 만들지 않는다.
- 사람이 쓴 것처럼 보이기 위해 고의로 오탈자, 비문과 사실 오류를 넣지 않는다.
- 최종 검사에서 소리 내어 읽었을 때 어색한 연결어, 같은 어미의 연속과 불필요한 요약 문장을 다듬는다.

실제 경험이 없는 경우에도 경험을 꾸며내지 않고 다음 방식으로 구체성을 만든다.

- 자사 데이터에 있는 실제 기능과 사용 절차
- 독자가 처한 구체적인 상황과 판단 기준
- 확인 가능한 조건을 적용한 예시
- 장점뿐 아니라 적용 범위, 예외와 주의점

## 3. 카드뉴스 워커

### 3.1 조사에서 얻은 원칙

Meta는 carousel을 여러 제품 소개, 한 제품의 기능별 설명, 순차적 이야기와 단계별 과정 설명에 적합한 형식으로 안내한다. 순서가 중요한 이야기라면 카드 순서를 유지해야 한다. Buffer의 대규모 게시물 분석과 실무 가이드는 첫 장의 hook, 다음 장으로 이어지는 이야기 구조, 읽기 쉬운 단순한 디자인과 저장·공유할 교육형 콘텐츠를 강조한다.

참고 자료:

- [Meta Carousel ads](https://www.facebook.com/business/ads/carousel-ad-format)
- [Buffer Instagram carousel guide](https://buffer.com/resources/instagram-reach/)
- [Buffer carousel performance analysis](https://buffer.com/resources/instagram-reach-engagement-analysis/)

### 3.2 Skill에 넣을 제작 규칙

- 먼저 `핵심 주제`, `독자가 얻는 것`, `전개 방식`을 결정한다.
- 전개 방식은 문제 해결, 체크리스트, 단계 안내, 비교, 오해 교정, 사례 설명 중 내용에 맞는 하나를 선택한다.
- 첫 장은 주제와 독자 이익을 바로 이해할 수 있는 제목으로 만든다. 모호한 감탄문과 낚시성 문구는 사용하지 않는다.
- 첫 장 hook은 `구체적인 결과`, `독자가 겪는 상황`, `피해야 할 실수`, `확인할 목록` 중 내용에 맞는 하나를 사용한다. 내용과 연결되지 않는 불안·호기심 자극은 사용하지 않는다.
- 2장 이후는 한 장마다 하나의 역할을 갖게 한다. 각 장이 앞 장을 반복하지 않아야 한다.
- 중간 장은 핵심 설명, 이유, 실제 적용 방법과 주의점 중 필요한 정보를 배치한다. 표지만 강하고 본문이 빈약한 결과를 허용하지 않는다.
- 마지막 장은 핵심 요약 또는 실제 다음 행동을 제시한다. 가짜 UI 버튼은 그리지 않는다.
- 장수는 내용을 충분히 설명하는 최소 개수로 정하며 1~5장 범위다. 항상 5장을 채우지 않는다.
- 카드끼리 같은 크기, 색 체계, 타이포그래피와 여백 규칙을 유지한다.
- 브랜드 주색은 강조와 체계에 사용하되 화면 전체를 한 색으로 채우지 않는다.
- 모바일에서 읽을 수 있도록 제목, 본문과 보조 정보의 위계를 분명히 한다.
- 정보가 많은 주제는 작은 글자를 무리하게 넣지 말고 핵심을 남긴다. 원본의 중요한 조건을 생략해 의미를 바꾸면 안 된다.
- 캡션은 카드 내용을 그대로 반복하지 않고 맥락, 핵심 요약과 CTA를 보완한다.

### 3.3 자체 품질검사

- 첫 장만 보고 대상과 얻을 수 있는 정보가 이해되는가
- 각 장의 역할이 다르고 순서가 자연스러운가
- 필요한 설명을 끝낸 시점에서 장수가 멈췄는가
- 한 장에 서로 다른 핵심 메시지가 과도하게 섞이지 않았는가
- 모바일 축소 화면에서도 핵심 문구가 읽히는가
- 저장하거나 공유할 구체적인 정보가 최소 하나 있는가
- hook에서 약속한 내용을 본문이 충분히 제공하는가

## 4. 블로그 워커

### 4.1 조사에서 얻은 원칙

Google Search Central은 검색 순위 조작보다 기존 독자에게 실제로 도움이 되는 사람 중심 콘텐츠, 독창적인 분석, 충분한 설명, 명확한 출처와 신뢰성을 우선하라고 안내한다. 사용자가 글을 읽은 뒤 목표를 달성할 만큼 배워야 하며, 다시 검색해야만 답을 얻는 불완전한 글을 경고한다. Google의 title 가이드는 페이지마다 고유하고 설명적인 간결한 제목을 권장하고 키워드 반복과 과장된 제목을 피하도록 한다. Nielsen Norman Group의 웹 글쓰기 연구는 사용자가 화면에서 정독보다 훑어보기를 하므로 짧은 문단, 설명적인 제목과 목록 구조가 필요하다고 설명한다.

참고 자료:

- [Google 사람 중심 콘텐츠 가이드](https://developers.google.com/search/docs/fundamentals/creating-helpful-content)
- [Google 생성형 AI 콘텐츠 가이드](https://developers.google.com/search/docs/fundamentals/using-gen-ai-content)
- [Google AI 검색 콘텐츠 최적화 가이드](https://developers.google.com/search/docs/fundamentals/ai-optimization-guide)
- [Google title link 가이드](https://developers.google.com/search/docs/appearance/title-link)
- [Google 이미지 SEO 가이드](https://developers.google.com/search/docs/appearance/google-images)
- [Nielsen Norman Group 웹 글쓰기](https://www.nngroup.com/articles/be-succinct-writing-for-the-web/)

### 4.2 Skill에 넣을 제작 규칙

- 예상 독자와 독자가 해결하려는 질문을 먼저 정의한다.
- 검색 의도를 정보 탐색, 비교, 문제 해결, 구매 전 확인 중 하나로 정하고 본문의 답변 범위를 그 의도에 맞춘다.
- 제목은 글의 실제 답변 범위를 구체적으로 설명하고 과장, 키워드 나열과 반복 문구를 피한다.
- 도입부는 독자의 상황, 글에서 답할 질문과 읽고 난 뒤 얻게 될 결과를 짧게 밝힌다.
- 본문은 하나의 명확한 H1 아래에 설명적인 H2·H3를 사용한다.
- 결론부터 제시한 뒤 이유, 근거, 방법, 예시와 주의점을 전개한다.
- 긴 연속 문단을 피하고 절차, 조건과 비교는 목록이나 표로 구조화한다.
- 원본 URL을 단순 요약하지 않고 브랜드가 제공할 수 있는 해석, 적용 방법과 의사결정 기준을 더한다.
- 핵심 검색어는 제목, 도입과 관련 절에서 문맥에 맞게 사용하되 같은 표현을 억지로 반복하지 않는다. 동의어와 실제 사용 용어를 자연스럽게 사용한다.
- 확인한 사실과 일반적인 조언을 구분한다. 근거가 없는 사례와 수치를 만들지 않는다.
- 정해진 글자 수를 채우지 않는다. 질문에 충분히 답한 시점에 마무리한다.
- 결론은 핵심 답변과 다음 행동을 정리하되 본문 전체를 다시 반복하지 않는다.
- meta title과 description은 본문과 일치해야 하며 각 결과마다 고유하게 만든다.
- 최종 HTML은 하나의 H1, 논리적인 H2·H3 순서, 의미 있는 `article`, `section`, `ul`, `ol`, `table`, `figure` 요소를 사용한다.
- 대표 이미지에는 주제를 설명하는 alt text를 함께 생성한다.
- 대표 이미지는 글의 주제를 구체적으로 보여주고 장식만을 위한 추상 이미지를 피한다.
- 본문에는 비교, 과정, 예시 또는 구조를 이해하는 데 실제로 도움이 되는 이미지 1~3장을 배치한다.
- 본문 이미지는 각각 구체적인 alt text를 가지며 장식용 이미지로 분량을 채우지 않는다.

### 4.3 자체 품질검사

- 독자가 다시 검색하지 않아도 핵심 질문의 답을 얻는가
- 원본을 바꿔 쓴 수준을 넘어 적용 가능한 가치가 있는가
- 제목이 실제 본문 범위를 정확히 설명하는가
- 제목 계층만 훑어도 글의 논리와 결론을 알 수 있는가
- 확인되지 않은 사실, 날짜, 가격과 성과 수치가 없는가
- 문단과 목록이 화면에서 빠르게 탐색 가능한가
- 본문 이미지가 주변 문단의 이해를 돕고 이미지 없이도 성립하는 정보 구조를 보완하는가
- 검색어를 제거해도 사람에게 유용하고 자연스러운 글로 남는가
- 브랜드가 실제로 겪지 않은 경험을 1인칭 사례처럼 쓰지 않았는가

## 5. 마케팅 소재 워커

### 5.1 조사에서 얻은 원칙

Meta는 고품질 시각물과 설득력 있는 메시지가 브랜드의 차별성을 전달해야 한다고 안내하며, 서로 다른 형식과 메시지의 creative diversification을 권장한다. Google Ads는 이미지에서 제품이나 서비스가 중심이 되어야 하고 흐림, 과한 필터, 가짜 버튼, 불명확한 합성과 과도한 텍스트를 피하라고 안내한다. 광고 제목과 설명은 단순하고 구체적이어야 하며, 일반적인 문구와 클릭베이트보다 실제 혜택·가격·프로모션 조건을 정확히 전달해야 한다.

참고 자료:

- [Meta Performance 5](https://www.facebook.com/business/ads/performance-marketing)
- [Meta Advantage+ creative](https://www.facebook.com/business/ads/meta-advantage-plus/creative)
- [Google 반응형 디스플레이 광고 제작 가이드](https://support.google.com/google-ads/answer/9823397)
- [Google Ads asset 원칙](https://support.google.com/google-ads/answer/7331111)

### 5.2 Skill에 넣을 제작 규칙

- 한 결과는 하나의 타깃, 하나의 핵심 혜택과 하나의 행동 목표에 집중한다.
- 광고의 첫 인상은 `대상 상황`, `구체적 혜택`, `제안 조건`, `문제 해결 결과` 중 하나를 선명하게 보여준다.
- 컨셉을 먼저 `대상 상황 → 문제 또는 욕구 → 제안 가치 → 행동`으로 정의한다.
- headline은 제품명만 쓰지 않고 사용자가 얻는 구체적인 이익이나 제안을 표현한다.
- body는 headline을 반복하지 않고 조건, 차별점 또는 신뢰 근거를 보완한다.
- CTA는 실제로 연결되는 행동과 일치하게 작성한다.
- 이미지에서 제품, 서비스 사용 장면 또는 핵심 결과가 분명한 초점이 되게 한다.
- 흐림, 과도한 필터, 왜곡, 의미 없는 넓은 빈 공간과 복잡한 collage를 피한다.
- 이미지 안의 텍스트는 시각적 핵심에 필요한 최소 문구만 사용한다. 긴 설명은 별도 카피에 둔다.
- 이미지 안에 실제 기능처럼 보이는 재생, 다운로드, 닫기와 자세히 보기 버튼을 넣지 않는다.
- 할인, 가격, 마감일과 독점 조건은 근거 데이터에 있을 때만 사용하고 구체적으로 쓴다.
- 결과를 여러 개 만들 때 색만 바꾸지 않는다. 문제 해결형, 혜택 강조형, 신뢰형처럼 메시지 가설을 다르게 한다.
- 카피는 실제 사람이 짧게 설명하는 문장처럼 쓰고, 근거 없는 최상급과 상투적인 AI 홍보 문구를 제거한다.
- 사용자가 지정한 `1:1`, `4:5`, `9:16` 비율에 맞춰 처음부터 구성하고 사후 crop을 전제로 만들지 않는다.

### 5.3 자체 품질검사

- 2초 안에 무엇을 누구에게 제안하는지 이해할 수 있는가
- 제품이나 서비스보다 장식 요소가 더 강하지 않은가
- headline, body와 CTA가 서로 다른 역할을 하는가
- 혜택과 조건이 근거 데이터와 일치하는가
- 여러 결과의 메시지 가설이 실제로 다른가
- 가짜 버튼, 클릭베이트와 읽기 어려운 작은 문구가 없는가
- 카피에서 제품명과 형용사를 빼도 구체적인 제안 가치가 남는가

## 6. CLI 전달 방식

각 워커에는 다음 Skill을 둔다.

- `brand-pilot-card-news-worker/.agents/skills/card-news-creator/SKILL.md`
- `brand-pilot-blog-worker/.agents/skills/blog-writer/SKILL.md`
- `brand-pilot-marketing-worker/.agents/skills/marketing-creative/SKILL.md`

각 Skill은 다음 구조를 갖는다.

1. 역할과 산출물 정의
2. 입력 JSON schema
3. 근거 사용과 금지 규칙
4. 기획 순서
5. 유형별 제작 규칙
6. 자체 품질검사
7. 최종 JSON schema와 파일명 계약

중앙 API의 job payload에는 사용자 입력과 함께 `skillVersion`을 기록한다. 워커 완료 결과에도 같은 버전을 돌려주어 어떤 제작 원칙으로 결과가 생성됐는지 추적한다. Skill 수정은 새 버전으로 올리고 이미 생성된 결과의 버전은 바꾸지 않는다.

워커 실행 중 일반 인터넷 검색은 하지 않는다. 시의성이 필요한 사실은 사용자가 제공한 URL과 크롤링된 자사 정보만 사용한다. 이 문서의 외부 자료는 제작 방법을 정하는 근거이며 고객 콘텐츠의 사실 근거로 사용하지 않는다.

## 7. 실제 상위 콘텐츠 특징 반영

공개 화면에 보인다는 이유만으로 상위 성과 콘텐츠라고 판단하지 않는다. 조회, 저장, 공유, 클릭 또는 전환처럼 비교 가능한 지표가 있는 결과만 고성과 레퍼런스로 분류한다.

### 카드뉴스

현재 Instagram 트렌드 탐색에서 저장한 같은 대표·세부 분야의 인기 carousel과 해당 브랜드가 발행한 카드뉴스 중 저장·공유·참여가 높은 결과를 우선한다. 워커에는 원문을 복사하지 않고 다음 분석값을 전달한다.

- 첫 장 hook 유형
- 전체 장수와 전개 방식
- 장별 정보 역할
- 텍스트 밀도와 시각 자료 비중
- 마지막 장의 요약·CTA 방식

### 블로그

상위 노출은 검색어와 시점에 따라 달라지므로 일반적인 상위 글 목록을 영구 규칙으로 저장하지 않는다. 주제별로 검토한 검색 결과가 있으면 검색어, 확인일, URL과 구조 분석을 레퍼런스 스냅샷으로 저장한다. 워커에는 다음 특징만 전달한다.

- 충족하려는 검색 의도
- 제목이 약속하는 답변 범위
- H2·H3 정보 구조
- 경쟁 글이 충분히 답하지 못한 질문
- 브랜드가 추가할 수 있는 실제 정보와 경험 근거

검색 결과를 제공하지 않은 생성에서도 Google의 사람 중심 콘텐츠와 기술 SEO 기본 규칙은 항상 적용한다.

### 마케팅 소재

공개 광고 라이브러리는 집행 여부를 보여줄 수 있지만 성과 순위를 보장하지 않는다. 따라서 공개 광고를 `상위 광고`로 자동 분류하지 않는다. 사용자가 저장한 광고 레퍼런스와 추후 연동된 자사 광고 성과 중 클릭·전환 지표가 확인된 소재를 사용한다. 워커에는 다음 특징만 전달한다.

- 타깃 상황과 메시지 가설
- headline, body, CTA의 역할 분리
- 제품·인물·결과 중 시각적 초점
- 혜택 또는 제안 조건의 표현 방식
- 비율별 정보 배치와 가독성

레퍼런스 분석값은 생성 prompt에 포함하지만 문장, 인물, 로고와 구도를 그대로 재사용하지 않는다.

## 8. 성과 기반 개선

향후 성과 데이터가 충분히 쌓이면 콘텐츠 유형 안에서만 비교한다.

- 카드뉴스: 저장, 공유, carousel 진행과 도달
- 블로그: 조회, 체류, 스크롤과 목표 행동
- 마케팅 소재: 노출 대비 클릭·전환

성과가 좋았던 결과의 문장을 그대로 재사용하지 않는다. `전개 방식`, `정보 밀도`, `메시지 가설`, `시각적 초점` 같은 구조적 특징만 레퍼런스 메타데이터로 제공한다. Skill 자체 변경은 반복적으로 확인된 패턴만 반영하고 단일 결과에 맞춰 자동 수정하지 않는다.

### 8.1 생성 전 품질 브리프

모든 생성 워커는 최종 산출물 전에 `content-quality.v1` 브리프를 만든다. 브리프에는 훅, 독자가 얻는 가치, 지금 다룰 이유, 구체 주장 2개 이상과 이를 뒷받침하는 근거 2개 이상을 포함한다. 확인 가능한 근거가 2개보다 적으면 결과물을 만들지 않고 분석 작업을 재시도한다. 재시도 한도를 소진하면 실패로 기록하며 근거 없는 문장을 보충하지 않는다.

### 8.2 비교 시점

발행 성과는 콘텐츠 특성과 함께 발행 후 `24h`, `72h`, `7d` 시점에 기록한다. 하루 한 번 실행되는 수집기의 오차를 허용하되 같은 게시물과 시점의 기록은 중복 저장하지 않는다. 저장 특성은 콘텐츠 형식, 주제, 관점, 훅, 주장 수와 근거 수이며 이후 동일 형식 안에서만 비교한다.
