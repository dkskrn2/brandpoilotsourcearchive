# 대시보드 성과 콘텐츠 설계

## 목적

자동 운영과 AI 콘텐츠 직접 게시를 구분하지 않고, 최근 30일 동안 실제 성과가 수집된 게시물을 한 목록에서 비교한다.

## 집계 기준

- 기존 `publish_queue`와 `content_performance_snapshots`를 사용한다.
- 최근 30일 내 게시 완료된 콘텐츠를 대상으로 한다.
- 최신 조회·노출 수가 있는 콘텐츠만 순위에 포함한다. 측정값 `0`은 유효한 성과로 유지한다.
- 조회·노출 수 내림차순으로 최대 10개를 표시한다.
- 자동 운영과 AI 콘텐츠 직접 게시의 출처는 순위 기준에 사용하지 않는다.

## 화면 구성

- 섹션 제목을 `성과가 좋았던 콘텐츠`로 변경한다.
- 목록 행 전체를 클릭 가능한 상세 진입점으로 제공한다.
- 상세 팝업은 이미지, 영상, HTML, 텍스트 결과물을 기존 게시 결과 미리보기 컴포넌트로 표시한다.
- 채널, 게시 유형, 게시일, 조회·노출 수를 함께 표시한다.
- 원본 게시 URL이 있으면 `원본 게시물 보기` 버튼을 제공한다.
- 결과물 데이터는 팝업을 열 때만 조회해 대시보드 초기 로딩을 늘리지 않는다.

## 오류 처리

- 결과물 조회 중에는 로딩 상태를 표시한다.
- 결과물 조회 실패 시 대시보드 목록은 유지하고 팝업 안에서 재시도할 수 있게 한다.
- 원본 게시 URL이 없으면 외부 링크 버튼을 표시하지 않는다.

## 검증

- 성과 미수집 항목은 순위에서 제외되고 측정값 `0`은 유지되는지 API 테스트로 확인한다.
- 목록 클릭 전에는 결과물 API가 호출되지 않고, 클릭 후 팝업에서 결과물과 게시 정보가 표시되는지 UI 테스트로 확인한다.
- 고객 UI 및 API 집중 테스트와 빌드를 실행한다.
