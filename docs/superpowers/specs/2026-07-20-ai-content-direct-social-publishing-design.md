# AI 콘텐츠 직접 SNS 게시 및 채널 로고 설계

작성일: 2026-07-20  
상태: 확정안  
범위: AI 콘텐츠 상세 화면의 결과물별 직접 게시, 게시 기록, SNS 로고 공통화

## 1. 목적

AI 콘텐츠 스튜디오에서 완성된 결과물을 게시관리 화면으로 이동시키지 않고, 결과물 상세 화면에서 연결된 SNS 계정으로 바로 게시할 수 있게 한다.

사용자는 결과물마다 게시할 채널과 게시 유형을 체크박스로 선택하고 `지금 게시`를 누른다. 화면 흐름은 즉시 게시지만, 서버는 기존 게시 큐와 게시 시도 기록을 계속 사용해 중복 게시, 재시도, 이력 조회를 관리한다.

채널을 식별하기 쉽도록 Instagram, Threads, X, LinkedIn, TikTok, YouTube의 공식 로고 자산을 로컬에 저장하고, 서비스에서 채널을 표현하는 공통 UI에 적용한다.

## 2. 확정된 사용자 경험

### 2.1 결과물별 게시 대상 선택

- 게시 UI는 AI 콘텐츠 생성 결과물 카드 또는 상세 영역 안에 둔다.
- 결과물별로 `SNS 게시 채널 및 유형` 영역을 제공한다.
- 채널은 Instagram, Threads, X, LinkedIn, TikTok, YouTube를 모두 표시한다.
- 한 결과물을 여러 채널 또는 여러 게시 유형으로 동시에 선택할 수 있다.
- 선택은 체크박스를 사용한다.
- 채널 행에는 로고, 채널명, 연결 계정, 연결 상태를 표시한다.
- 연결된 채널은 `@계정명` 또는 플랫폼 계정명을 표시한다.
- 연결되지 않은 채널은 `OAuth 게시 계정 미연결`이라고 표시한다. `연결되지 않음`만 단독으로 표시하지 않는다.
- `OAuth 연결 계정만 표시됩니다` 안내 문구는 사용하지 않는다.
- 연결되지 않은 채널에도 `연결하기` 버튼을 제공한다.
- 실제 OAuth 시작 URL이 없는 채널은 버튼 클릭 시 `연결 준비 중` 안내만 표시하며 외부 이동은 하지 않는다.
- 현재 결과물과 호환되지 않는 게시 유형은 비활성화하고 구체적인 이유를 함께 표시한다. 예: `영상 결과 필요`.

### 2.2 즉시 게시

- 최종 확인 모달은 사용하지 않는다.
- 버튼 문구는 선택 수를 반영한다. 예: `선택한 2곳에 지금 게시`.
- 버튼 클릭 시 서버에 선택한 게시 대상을 한 번에 요청한다.
- 서버는 각 대상을 독립적으로 검증하고 게시한다.
- 한 대상이 실패해도 나머지 대상 게시를 계속한다.
- 결과물 화면에는 대상별로 `게시 준비 중`, `게시 중`, `게시 완료`, `게시 실패` 상태를 표시한다.
- 실패한 대상만 다시 시도할 수 있게 한다.
- 게시 성공 후에도 게시관리 화면과 게시 이력에는 동일한 기록이 남는다.

## 3. 콘텐츠 유형별 게시 호환성

게시 대상은 콘텐츠 유형과 실제 산출물 형식으로 결정한다. UI와 서버가 같은 호환성 표를 사용해야 한다.

| AI 콘텐츠 | 게시 채널/유형 | 지원 조건 | 처리 방식 |
| --- | --- | --- | --- |
| 카드뉴스 | Instagram 게시글 | 이미지 1장 이상 | 1장이면 단일 게시물, 2장 이상이면 캐러셀 |
| 카드뉴스 | Instagram 스토리 | 이미지 1장 이상 | 첫 번째 이미지를 스토리 1장으로 게시 |
| 카드뉴스 | Instagram 릴스 | 미지원 | 영상 산출물이 없으므로 비활성화 |
| 마케팅 소재 | Instagram 게시글 | 이미지 1장 이상 | 첫 번째 이미지를 단일 게시물로 게시 |
| 마케팅 소재 | Instagram 스토리 | 이미지 1장 이상 | 첫 번째 이미지를 스토리 1장으로 게시 |
| 마케팅 소재 | Instagram 릴스 | 미지원 | 영상 산출물이 없으므로 비활성화 |
| 블로그 | Instagram 전체 | 미지원 | HTML 산출물은 Instagram 게시 자산으로 사용하지 않음 |
| 모든 현재 AI 콘텐츠 | TikTok/YouTube 영상 | 미지원 | 영상 산출물이 없으므로 비활성화 |

Threads, X, LinkedIn은 OAuth 및 게시 어댑터를 붙이기 전까지 채널 행과 연결 상태만 제공한다. 향후 이미지 또는 링크 게시 어댑터가 준비되면 동일한 호환성 표에 유형을 추가한다.

이미지 비율은 생성 단계에서 사용자가 선택한 결과를 그대로 사용한다. 이 기능에서 크롭이나 리사이즈를 수행하지 않는다. 플랫폼이 허용하지 않는 비율이면 게시 전에 `delivery_format_asset_mismatch`로 차단한다.

## 4. 화면 구성

### 4.1 연결된 채널

연결된 채널 행은 다음 순서로 구성한다.

1. 채널 로고와 채널명
2. 연결 계정명과 `OAuth 게시 계정 연결됨`
3. 지원되는 게시 유형 체크박스
4. 호환되지 않는 유형과 비활성 사유

Instagram 예시:

- `Instagram`
- `@growthline352`
- `게시글` 체크박스
- `스토리` 체크박스
- `릴스` 비활성 체크박스와 `영상 결과 필요`

### 4.2 연결되지 않은 채널

연결되지 않은 채널 행은 다음을 표시한다.

- 채널 로고와 채널명
- `OAuth 게시 계정 미연결`
- `연결하기` 버튼

연결되지 않은 채널에는 게시 유형 체크박스를 노출하지 않는다. 사용자가 `연결하기`를 통해 채널을 연결한 뒤 게시 유형을 선택하게 한다.

### 4.3 게시 상태

게시 요청 후 별도 모달을 열지 않고 현재 결과물 영역에서 상태를 갱신한다.

- 요청 전: 선택 체크박스와 `지금 게시`
- 요청 직후: 선택 항목을 잠그고 로딩 상태 표시
- 일부 성공: 성공 항목은 게시 URL, 실패 항목은 오류와 `다시 시도`
- 전체 성공: 게시 완료 시각과 게시 URL 표시

## 5. API 계약

기존 엔드포인트를 다중 대상 요청으로 확장한다.

```http
POST /brands/:brandId/ai-content/outputs/:outputId/publish
```

요청 본문:

```json
{
  "idempotencyKey": "client-generated-uuid",
  "targets": [
    {
      "channel": "instagram",
      "deliveryFormat": "instagram_feed_carousel"
    },
    {
      "channel": "instagram",
      "deliveryFormat": "instagram_story"
    }
  ]
}
```

응답은 대상별 결과를 반환한다.

```json
{
  "outputId": "output-uuid",
  "targets": [
    {
      "channel": "instagram",
      "deliveryFormat": "instagram_feed_carousel",
      "queueId": "queue-uuid",
      "status": "published",
      "publishedUrl": "https://www.instagram.com/p/..."
    },
    {
      "channel": "instagram",
      "deliveryFormat": "instagram_story",
      "queueId": "queue-uuid",
      "status": "scheduled",
      "errorCode": null
    }
  ]
}
```

서버 검증 순서:

1. 로그인 사용자가 해당 워크스페이스와 브랜드에 접근 가능한지 확인
2. 결과물이 해당 브랜드 소유이며 `completed`인지 확인
3. 요청에 같은 채널·게시 유형 조합이 중복되지 않았는지 확인
4. 결과물 manifest와 게시 유형이 호환되는지 확인
5. 활성화된 채널과 유효한 OAuth credential이 있는지 확인
6. 기존 게시 기록 또는 같은 멱등성 키가 있는지 확인
7. 게시 큐와 대상별 채널 출력을 생성
8. 대상별 즉시 게시 시도를 순차 실행

## 6. 게시 처리 구조

화면에서는 직접 게시하지만 기존 게시 인프라를 우회하지 않는다.

```text
AI 콘텐츠 결과
→ 대상별 channel_outputs 생성
→ 대상별 publish_queue 생성(status=scheduled, scheduled_for=now)
→ 트랜잭션 커밋
→ 중앙 API가 대상별 publishQueueItem을 순차 호출
→ publish_attempts 및 외부 게시 ID 기록
→ 결과물 화면에 대상별 상태 반환
```

즉시 게시 시도 전에 큐를 먼저 저장한다. 요청 연결이 끊기거나 API 프로세스가 중단되어도 `scheduled` 상태가 남아 기존 로컬 스케줄러의 `runDuePublishing`이 다음 주기에 처리한다.

동시 외부 호출은 하지 않는다. 한 요청의 게시 대상은 순차 처리해 플랫폼 rate limit과 중복 게시 위험을 줄인다.

## 7. AI 콘텐츠 manifest 변환 규칙

AI 콘텐츠 결과의 원본 파일은 수정하지 않는다. 게시 대상별 adapter가 기존 manifest를 읽어 게시 자산을 선택한다.

- Instagram 게시글 캐러셀: 카드뉴스의 모든 이미지 사용
- Instagram 게시글 단일: 마케팅 소재의 첫 번째 이미지 사용
- Instagram 스토리: 첫 번째 이미지 사용
- Instagram 릴스: `video` 자산이 있을 때만 허용

스토리 게시를 위해 이미지를 새로 크롭하거나 다시 생성하지 않는다. Instagram publisher가 AI 콘텐츠 manifest의 첫 번째 이미지 URL을 스토리 자산으로 해석할 수 있게 manifest reader를 확장한다.

## 8. 데이터 모델과 멱등성

현재 `channel_outputs.ai_content_generation_output_id`는 결과물당 한 행만 허용한다. 이를 결과물·채널·게시 유형 조합별 한 행으로 변경한다.

고유 기준:

```text
(ai_content_generation_output_id, channel, delivery_format)
```

구조:

- AI 결과물 하나당 `content_topics`와 `master_drafts`는 한 번만 생성
- AI 결과물 하나당 게시 묶음 `topic_publish_groups`는 한 번만 생성
- 선택한 채널·게시 유형마다 `channel_outputs`와 `publish_queue`를 각각 생성
- `publish_queue.idempotency_key`는 AI 결과물 ID, 채널, 게시 유형을 포함해 구성
- 같은 결과물·채널·게시 유형을 다시 요청하면 새 외부 게시물을 만들지 않고 기존 게시 상태를 반환
- 사용자가 실패 항목을 재시도하면 기존 큐를 재사용하고 새 큐를 만들지 않음

## 9. 오류와 부분 실패

표준 오류 코드는 다음과 같다.

- `ai_content_output_not_found`: 결과물이 없음
- `ai_content_output_not_completed`: 생성이 완료되지 않음
- `duplicate_publish_target`: 요청 본문에 같은 대상이 중복됨
- `ai_content_publish_target_unsupported`: 현재 콘텐츠 유형이 대상을 지원하지 않음
- `channel_oauth_not_connected`: 활성 OAuth 게시 계정이 없음
- `delivery_format_asset_mismatch`: 결과물 자산과 게시 유형이 맞지 않음
- `publish_failed`: 외부 게시 실패

대상별 실패를 전체 요청 실패로 합치지 않는다. 서버는 성공한 대상, 재시도 예정 대상, 실패한 대상을 모두 응답한다. 외부 게시 성공 후 DB 최종 기록이 실패한 경우 기존 `publish_attempts` 성공 체크포인트를 기준으로 큐를 복구하며 외부 게시를 다시 호출하지 않는다.

## 10. SNS 로고 자산

### 10.1 저장 방식

상표 로고는 이미지 생성 모델로 만들지 않는다. 공식 브랜드 리소스 또는 검증된 아이콘 원본을 사용한다.

저장 경로:

```text
apps/customer-ui/public/assets/channels/
  instagram.svg
  threads.svg
  x.svg
  linkedin.svg
  tiktok.svg
  youtube.svg
  NOTICE.md
```

자산 우선순위:

1. 각 플랫폼 공식 브랜드 리소스의 SVG 또는 PNG
2. 공식 SVG가 제공되지 않으면 버전을 고정한 Simple Icons SVG

`NOTICE.md`에는 채널별 출처 URL, 취득일, 원본 라이선스 또는 상표 안내를 기록한다. 로고 색상과 형태를 임의로 변형하지 않는다.

### 10.2 공통 컴포넌트

`ChannelLogo` 컴포넌트가 채널 코드와 로컬 자산 경로를 매핑한다. 화면별로 SVG를 복사하거나 직접 그리지 않는다.

적용 범위:

- AI 콘텐츠 결과물의 SNS 게시 영역
- 채널 연결 페이지
- 관리자 채널 페이지
- 온보딩 채널 연결 항목
- 게시관리 카드와 상세 팝업의 채널 표시
- 대시보드의 채널별 성과 표시
- Instagram 트렌드 화면에서 채널을 나타내는 위치

텍스트 채널명이 로고 옆에 있으면 로고의 대체 텍스트는 빈 문자열로 두어 스크린리더 중복 읽기를 막는다.

## 11. 테스트 범위

### 11.1 API와 DB

- 결과물·채널·게시 유형 조합별 여러 `channel_outputs` 생성
- 같은 조합 재요청 시 기존 결과 반환
- 다른 게시 유형은 별도 게시 허용
- 연결되지 않은 채널 차단
- 완료되지 않은 결과물 차단
- 카드뉴스 게시글/스토리 자산 선택
- 마케팅 소재 게시글/스토리 자산 선택
- 블로그와 영상 전용 유형 차단
- 일부 대상 실패 시 나머지 대상 계속 처리
- 외부 게시 성공 후 DB 기록 실패 시 중복 외부 호출 방지
- API 요청 중단 후 스케줄러 복구

### 11.2 프론트엔드

- 모든 채널 행 표시
- 연결된 Instagram 계정명 표시
- 연결되지 않은 채널의 구체적인 상태와 `연결하기` 버튼 표시
- 콘텐츠 유형별 체크박스 활성/비활성
- 복수 선택과 선택 수 버튼 문구
- 최종 확인 모달 없이 게시 시작
- 대상별 게시 상태와 실패 재시도
- 로고 누락 시 레이아웃이 깨지지 않는 fallback

### 11.3 E2E

- 카드뉴스 결과에서 Instagram 게시글 1건 게시
- 카드뉴스 결과에서 게시글과 스토리를 동시에 선택해 각각 기록
- 마케팅 소재를 Instagram 단일 게시물로 게시
- 연결되지 않은 채널에서 게시 체크박스가 나오지 않는지 확인
- 동일 대상을 두 번 요청해 외부 게시가 한 번만 수행되는지 확인

CI에서는 Meta 외부 게시 호출을 mock 처리한다. 실제 Instagram 게시 검증은 연결된 테스트 계정으로 별도 smoke test에서 수행한다.

## 12. 이번 범위에서 제외

- Threads, X, LinkedIn, TikTok, YouTube의 실제 OAuth 설정
- 미연결 채널의 실제 OAuth 시작 URL
- 영상 생성 또는 정지 이미지를 영상으로 변환하는 기능
- 예약 게시 시간 선택
- 게시 전 최종 확인 모달
- 결과물 편집기
- 게시를 위해 이미지 크롭 또는 리사이즈

## 13. 적용 순서

1. 결과물·채널·게시 유형 호환성 계약과 DB 고유 기준 변경
2. 다중 대상 게시 API와 즉시 게시·스케줄러 복구 연결
3. AI 콘텐츠 manifest의 게시 유형별 adapter 구현
4. 결과물별 체크박스 UI와 대상별 상태 표시
5. 공식 SNS 로고 자산과 `ChannelLogo` 공통 컴포넌트 추가
6. 서비스 내 기존 채널 표시를 공통 로고 컴포넌트로 교체
7. 단위·통합·E2E 테스트 후 실제 Instagram smoke test
