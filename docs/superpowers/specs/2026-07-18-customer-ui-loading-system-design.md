# 고객 UI 공통 로딩 체계 설계

작성일: 2026-07-18  
상태: 확정

## 1. 목적

고객 화면의 비동기 조회와 명령 처리 상태를 일관되게 표시한다. 요청이 진행 중인데 이전 오류가 먼저 보이거나, 단순 문장만 표시되어 화면이 멈춘 것처럼 느껴지는 문제를 제거한다.

이번 범위에는 Instagram 트렌드 검색 UI 개선과 추천·최근 해시태그 즉시 검색을 포함한다.

## 2. 기본 원칙

1. 최초 조회 중에는 오류 문구 대신 화면 구조를 반영한 스켈레톤을 표시한다.
2. 기존 데이터가 있는 재조회에서는 데이터를 지우지 않고 반투명 로딩 오버레이를 표시한다.
3. 요청이 최종 실패한 뒤에만 오류 안내를 표시한다.
4. 중복 실행 위험이 있는 입력과 버튼은 처리 중 비활성화한다.
5. 콘텐츠 생성, 게시 대기, 재생성처럼 서버에서 오래 유지되는 업무 상태는 로딩 UI로 바꾸지 않는다.
6. 로딩 상태는 `role="status"`, 오류는 `role="alert"`를 사용하며 회전 애니메이션은 모션 감소 설정을 존중한다.

## 3. 공통 컴포넌트

### 3.1 PageSkeleton

- 페이지 최초 진입에 사용한다.
- 제목, 요약 지표, 표 또는 카드 영역의 전체 골격을 표현한다.
- 모든 페이지를 같은 빈 박스로 만들지 않고 페이지 유형별 조합을 허용한다.

### 3.2 ListSkeleton

- 표, 대화 목록, 게시 목록에 사용한다.
- 실제 목록과 같은 열 수와 대략적인 행 높이를 유지한다.
- 모바일에서는 가로 넘침 없이 한 열 구조로 축소한다.

### 3.3 CardSkeleton

- 트렌드 미디어, 성과 카드처럼 시각적 콘텐츠 목록에 사용한다.
- 이미지 비율과 텍스트 줄 수를 고정해 로딩 완료 시 레이아웃 이동을 줄인다.

### 3.4 LoadingOverlay

- 기존 데이터가 있는 재조회에 사용한다.
- 콘텐츠는 읽을 수 있게 유지하되 조작은 막는다.
- 중앙에 작은 스피너와 짧은 상태 문구를 표시한다.

### 3.5 InlineSpinner

- 저장, 업로드, 새로고침, 수동 응답 등 명령 버튼에 사용한다.
- 버튼 폭이 변하지 않도록 안정적인 크기를 사용한다.
- 기존 명령명을 유지하고 앞에 작은 스피너를 표시한다.

## 4. 화면별 적용

### 4.1 Instagram 트렌드 탐색

- 입력창과 검색 버튼을 하나의 검색바로 구성한다.
- 검색 버튼은 아이콘과 `검색` 명령을 함께 표시한다.
- 추천 및 최근 해시태그를 누르면 입력값을 즉시 바꾸고 검색을 시작한다.
- 검색 중에는 검색 버튼, 추천 태그, 최근 태그를 비활성화한다.
- 최초 검색은 결과 영역에 `CardSkeleton`을 표시한다.
- 기존 결과를 새로고침할 때는 결과 위에 `LoadingOverlay`를 표시한다.
- 검색 중에는 `트렌드 탐색 상태` 오류를 표시하지 않는다.
- API 요청이 끝까지 실패하면 기존 결과는 유지하고 그때 오류를 표시한다.

### 4.2 대시보드

- 최초 조회에는 요약 지표, 차트, 채널 표를 닮은 `PageSkeleton`을 표시한다.
- 성과 재조회 기능이 추가될 경우 기존 지표 위에 `LoadingOverlay`를 사용한다.

### 4.3 브랜드 설정

- 최초 조회에는 입력 필드와 설정 영역을 닮은 폼 스켈레톤을 표시한다.
- 저장 중에는 저장 버튼에 `InlineSpinner`를 표시하고 중복 제출을 막는다.
- 일부 보조 API 실패는 해당 섹션에만 오류를 표시한다.

### 4.4 결제 및 구독

- 결제 요약, 청구 이력, 결제 정보 영역을 닮은 `PageSkeleton`을 표시한다.
- 결제 API 최종 실패 후에만 오류 안내를 표시한다.

### 4.5 게시 관리 상세

- 결과물 팝업에서 이미지·영상·HTML 미리보기와 업로드 정보 영역에 스켈레톤을 표시한다.
- 다운로드 또는 저장 중에는 해당 버튼에 `InlineSpinner`를 표시한다.

### 4.6 DM 자동답변

- 대화 목록은 `ListSkeleton`, 대화 상세는 말풍선 형태 스켈레톤을 사용한다.
- Wiki 상태와 지식 데이터 조회는 해당 패널 안에서만 스켈레톤을 표시한다.
- 수동 응답, 지식 업로드, Wiki 재생성은 버튼 `InlineSpinner`와 중복 실행 차단을 사용한다.

### 4.7 온보딩 및 채널

- 온보딩 상태와 채널 연결 목록의 최초 조회에도 공통 목록 스켈레톤을 적용한다.
- 인증 필요, 권한 부족, 만료 상태는 로딩이 아니므로 기존 경고를 유지한다.

## 5. 상태 흐름

```text
idle
  -> initial_loading -> success
  -> initial_loading -> error

success
  -> refreshing -> success
  -> refreshing -> error_with_previous_data
```

- `initial_loading`: 페이지 또는 섹션 스켈레톤
- `refreshing`: 기존 데이터 + 로딩 오버레이
- `error`: 오류 안내와 재시도 명령
- `error_with_previous_data`: 기존 데이터 유지 + 비차단 오류 안내

## 6. 스타일

- 기존 Brand Pilot 색상 토큰과 8px 이하 모서리 체계를 유지한다.
- 스켈레톤은 중립 회색 계열을 사용하고 브랜드 주색으로 화면을 덮지 않는다.
- 애니메이션은 느린 좌우 이동 또는 명도 변화 한 종류만 사용한다.
- `prefers-reduced-motion: reduce`에서는 애니메이션을 제거한다.
- 스켈레톤은 실제 콘텐츠 크기를 대략적으로 반영해 완료 시 화면 이동을 줄인다.

## 7. 오류 처리

- 검색 또는 조회 시작 시 이전 오류를 지운다.
- 요청 중에는 오류와 로딩을 동시에 표시하지 않는다.
- 최종 실패 시 사용자에게 필요한 짧은 오류만 표시한다.
- 내부 오류 코드, 토큰, API 응답 전문은 화면에 노출하지 않는다.
- 재시도 가능한 화면에는 명확한 재시도 버튼을 제공한다.

## 8. 테스트

- 추천 태그 클릭 한 번으로 입력값 변경과 검색 API 호출이 함께 발생한다.
- 최근 태그도 같은 동작을 한다.
- 검색 중 태그와 검색 버튼이 비활성화된다.
- 검색 중 이전 오류가 숨겨지고 스켈레톤 또는 오버레이만 표시된다.
- 최종 실패 후 오류가 표시되고 기존 결과가 유지된다.
- 공통 로더가 접근성 상태를 제공한다.
- 각 주요 페이지의 최초 로딩 스켈레톤을 컴포넌트 테스트로 검증한다.
- 트렌드 검색은 데스크톱·모바일 E2E로 검증한다.

## 9. 제외 범위

- 콘텐츠 생성 워커와 게시 워커의 업무 상태 변경
- API 재시도 정책 변경
- 페이지 데이터 캐시 도입
- 로딩 애니메이션을 위한 외부 라이브러리 추가
