# Brand Pilot 통합형 가이드 설계

작성일: 2026-07-18

상태: 확정

선택안: C. 전역 도움말 + 화면별 코치마크 + OAuth 상세 가이드

## 목적

처음 사용하는 고객이 별도 매뉴얼을 먼저 읽지 않아도 현재 화면의 역할과 다음 행동을 이해하게 한다. 가이드는 작업을 가리지 않도록 사용자가 요청할 때만 연다.

## 구성

1. 사이드바의 `도움말` 버튼은 전체 가이드 서랍을 연다.
2. `PageHeader`가 있는 화면은 `화면 안내` 버튼을 공통 제공한다.
3. 화면 안내는 실제 UI를 가리키는 짧은 코치마크로 실행한다.
4. OAuth처럼 외부 서비스 설정이 필요한 절차는 서랍 안 체크리스트와 외부 이동 버튼으로 제공한다.
5. 짧은 아이콘 설명만 툴팁을 사용한다. 긴 절차를 툴팁에 넣지 않는다.

## 노출 정책

- 자동 실행하지 않는다.
- 사용자가 닫은 투어를 다시 자동으로 열지 않는다.
- 경로가 바뀌면 현재 화면에 맞는 문서를 표시한다.
- 찾을 대상이 없는 코치마크 단계는 건너뛰고 다음 유효 단계로 이동한다.
- 모바일에서는 도움말 서랍과 코치마크를 화면 폭에 맞춰 하단 중심으로 배치한다.

## 화면별 범위

- 대시보드: 최근 30일 기준, 운영 흐름, 채널 성과
- AI 콘텐츠: 콘텐츠 유형, 생성 시작, 생성 이력과 사용량
- 게시 관리: 상태 필터, 결과 미리보기, 검토와 다운로드
- 소스: 참고 URL, 크롤링 이력, 주제표 업로드
- 트렌드 탐색: Meta 연결, 해시태그 검색, 레퍼런스 저장
- 채널: OAuth 연결, 인증 후 활성화, 연결 상태, DM 준비 상태
- DM 자동답변: 대화, 수동 답변, 자사 정보, 중단·재개
- 브랜드 설정: 기본 프로필, 자사 URL, 분야, 생성 기준, 자동 승인
- 고객센터: 문의 접수, 처리 상태, 답변 확인
- 온보딩: 남은 설정과 이동 순서
- 관리자 채널: 고객 문의 처리

## 채널 OAuth 가이드

Instagram은 다음 체크리스트를 제공한다.

1. Instagram 전문 계정 확인
2. Facebook Page 연결
3. Meta Business 자산 연결 및 권한 확인
4. Brand Pilot에서 `Meta OAuth 연결` 실행
5. 연결 완료 후 채널 활성화

OAuth 비밀값, 계정 비밀번호, 액세스 토큰을 고객 화면에서 직접 입력받지 않는다.

## 접근성

- 도움말 서랍은 `dialog`와 `aria-modal`을 사용한다.
- 닫기, 이전, 다음 버튼은 텍스트 이름을 제공한다.
- `Escape`로 서랍과 투어를 닫을 수 있다.
- 코치마크 대상에는 시각적 강조뿐 아니라 제목과 설명을 함께 표시한다.
- 움직임 감소 설정에서는 전환 애니메이션을 최소화한다.

## 제외

- 페이지 접속 시 자동 팝업
- 모든 아이콘에 긴 설명 추가
- 외부 서비스 설정 전체를 한 팝업에 몰아넣기
- 가이드 완료 여부를 서버 DB에 저장하는 기능
