# AI 콘텐츠 생성 스튜디오 UI 설계

작성일: 2026-07-18  
상태: 화면 우선 구현 확정  
참고: ZET 제품 분석·광고 생성 플로우, Brand Pilot 기존 콘텐츠 생성·게시 관리 구조

## 1. 목표

Brand Pilot 고객이 긴 프롬프트를 직접 작성하지 않고 제품과 주제 정보를 입력한 뒤 AI가 제안한 타깃과 소구점을 선택해 카드뉴스, 블로그, 마케팅 이미지 소재를 생성할 수 있는 고객용 화면을 추가한다.

이번 범위는 React 화면, 라우팅, 단계 이동, 입력 상태, mock 결과와 사용량 표시까지다. 실제 LLM 호출, 이미지 생성, API, DB, 사용량 차감, 결제 연동은 포함하지 않는다.

## 2. 제품 원칙

- ZET의 `제품 분석 → 타깃·소구점 선택 → 레퍼런스 선택 → 생성 지시 → 결과 관리` 흐름을 따른다.
- 사용자가 모든 내용을 직접 작성하지 않는다. AI가 분석과 후보를 제안하고 사용자는 선택·수정한다.
- 생성 모델은 서비스가 고정하며 고객에게 모델 선택 UI를 제공하지 않는다.
- 직접 편집기는 제공하지 않는다.
- 광고 영상은 제공하지 않는다.
- 제품 URL은 선택값이다. 기존 자사 정보만으로도 생성할 수 있다.
- ZET의 편집기, 영상 변환, 규격 최적화처럼 Brand Pilot 운영 범위와 맞지 않는 도구는 복제하지 않는다.
- 자동 운영의 하루 최대 4개 주제 정책과 AI 콘텐츠 생성 사용량은 분리한다.

## 3. 콘텐츠 유형

### 3.1 카드뉴스

- Instagram 정방형 이미지 1~5장
- 게시 캡션
- 해시태그와 CTA

### 3.2 블로그

- 제목
- 요약
- 목차와 본문
- 메타 제목과 메타 설명
- 대표 이미지 1장

### 3.3 마케팅 소재

- 광고 컨셉
- 헤드라인, 본문 카피, CTA
- 광고 이미지
- 정적 이미지 규격 `1:1`, `4:5`, `9:16` 중 하나

광고 영상과 Google·Naver의 세부 배너 규격은 후속 범위다. 이번 화면은 유료 광고 플랫폼에 직접 게시하는 기능을 제공하지 않는다.

## 4. 정보 구조와 경로

사이드바에 `AI 콘텐츠 생성` 메뉴를 추가한다.

| 경로 | 역할 |
|---|---|
| `/ai-content` | 콘텐츠 유형 선택, 오늘 사용량, 성과 우수 콘텐츠, 최근 생성 결과 |
| `/ai-content/new` | 단계형 콘텐츠 생성 플로우 |
| `/ai-content/:generationId` | 생성 진행 상태와 결과 상세 |

`/ai-content`의 첫 화면에는 다음을 표시한다.

- 카드뉴스, 블로그, 마케팅 소재 시작 버튼
- 오늘 생성 사용량
- 오늘 신규 다운로드 사용량
- 진행 중인 생성 작업과 단계별 상태
- 해당 브랜드가 직접 발행한 콘텐츠 중 채널·형식 안에서 성과가 좋은 항목
- 최근 생성 결과와 상태, 유형별 필터
- 실패·부분 실패 결과의 상세 이동과 재시도

성과 우수 콘텐츠는 서로 다른 채널의 조회수, 좋아요와 댓글을 하나의 점수로 섞지 않는다. `Instagram 카드뉴스 조회수 상위`, `Instagram 릴스 조회수 상위`처럼 동일 채널·형식 안에서 비교하고, 성과 데이터가 없거나 오래됐으면 해당 상태를 표시한다.

## 5. 단계형 생성 플로우

### Step 1. 유형 선택

- 카드뉴스, 블로그, 마케팅 소재 중 하나를 선택한다.
- 예상 결과 규격과 예상 생성 시간을 표시한다.
- 오늘 남은 생성 한도를 표시한다.
- 한도에 도달한 유형은 선택할 수 없고 제한 사유와 초기화 시각을 표시한다.

### Step 2. 제품·주제 분석

- 등록된 자사 정보를 기본 근거로 사용한다.
- 제품 URL을 선택 입력할 수 있다.
- 제품 이미지와 분석할 페이지를 선택할 수 있다.
- 페이지에서 발견한 이미지와 직접 올린 이미지 중 최대 5개를 분석 이미지로 선택한다.
- 분석 대기, 분석 중, 분석 완료, 분석 실패 상태를 표시한다.
- 분석 완료 후 확인된 사실, 특징, 가격, 혜택과 근거를 요약한다.
- 기본 화면에는 짧은 요약을 표시하고 `분석 상세 보기`에서 USP, 고객 문제, 구매 동기, 대안의 장단점과 확인 근거를 펼쳐본다.
- 분석 근거 URL은 내부 확인용으로만 사용하고 생성 결과물의 본문이나 이미지에는 노출하지 않는다.

### Step 3. 타깃·소구점

- `AI 추천`과 `저장한 항목`을 전환해서 본다.
- AI가 타깃 고객 후보 3개를 제안한다.
- 각 후보에 상황, 문제, 구매 동기를 함께 표시한다.
- 사용자는 타깃 하나를 선택하거나 직접 추가·수정할 수 있다.
- AI 추천 또는 직접 입력한 타깃을 현재 브랜드의 타깃 목록에 저장할 수 있다.
- 핵심 소구점 하나와 보조 소구점 최대 두 개를 선택한다.
- AI 추천 또는 직접 입력한 소구점을 현재 브랜드의 소구점 목록에 저장할 수 있다.
- 저장한 타깃과 소구점은 다음 콘텐츠 생성에서 다시 불러온다.
- 저장한 소구점 자체에는 핵심·보조 역할을 고정하지 않고 생성할 때마다 역할을 지정한다.
- AI 분석 근거를 펼쳐 볼 수 있다.

저장 항목 규칙은 다음과 같다.

- 저장 범위는 사용자 개인이 아니라 현재 브랜드다.
- 타깃은 이름, 상황, 문제, 구매 동기를 저장한다.
- 소구점은 제목, 설명과 근거 유형을 저장한다.
- 목록은 최근 사용과 사용 빈도를 기준으로 정렬한다.
- 현재 생성에는 선택 당시 값을 스냅샷으로 저장한다.
- 저장 항목을 나중에 수정하거나 삭제해도 과거 생성 결과는 변경하지 않는다.
- 같은 브랜드 안에서 정규화한 이름과 내용이 같은 항목은 중복 저장하지 않는다.

### Step 4. 레퍼런스

- 기존 업로드·생성 콘텐츠, API로 수집한 콘텐츠와 직접 업로드한 레퍼런스를 하나의 라이브러리에서 탐색한다.
- ZET의 레퍼런스 화면을 참고해 `분야별`, `소구점별`, `직접 찾기`, `내 보관함`, `레퍼런스 올리기` 탭을 제공한다.
- `분야별`에서는 대표 분야를 먼저 고르고 해당 대표 분야에 속한 세부 분야를 선택한다.
- 콘텐츠 형식은 카드뉴스, 단일 이미지, 릴스, 블로그와 마케팅 소재로 필터한다.
- 출처는 자체 콘텐츠, API 수집, 저장한 트렌드와 직접 업로드로 필터한다.
- 내 발행 콘텐츠 중 같은 채널·형식 안에서 성과가 우수한 항목은 성과 지표와 함께 표시한다.
- 참고 항목을 최대 5개 선택한다.
- 레퍼런스 선택은 필수가 아니며 선택하지 않아도 다음 단계로 이동할 수 있다.
- 데스크톱에서는 선택한 레퍼런스를 우측 고정 영역에, 모바일에서는 하단 요약 영역에 표시한다.
- 선택한 항목은 확대 미리보기, 제거와 순서 변경을 제공한다.
- 레퍼런스는 복제 대상이 아니라 구성, 톤, 정보 표현 참고 자료로 사용한다.

레퍼런스 분류 규칙은 다음과 같다.

- 자체 생성 결과는 생성 당시 브랜드의 대표·세부 분야와 콘텐츠 형식을 상속한다.
- API 수집 콘텐츠는 검색에 사용한 해시태그와 해당 브랜드의 대표·세부 분야를 기본값으로 사용한다.
- 직접 업로드할 때 대표 분야, 세부 분야, 콘텐츠 형식과 선택 소구점을 입력한다.
- 기존 항목에 분류 정보가 없으면 `미분류`로 표시하고 레퍼런스 목록에서 필터할 수 있다.
- 마케팅 레퍼런스가 부족한 초기에는 `레퍼런스 올리기`를 기본 진입점으로 노출한다.

### Step 5. 생성 지시

- 목적
- 강조할 내용
- CTA
- 추가 지시
- 제품 사진
- 선택 인물 사진
- 크기·비율 참고 이미지
- 추가 시각 참고 이미지와 PDF

생성 모델 선택 UI는 표시하지 않는다.

유형별 제작 개수는 다음과 같다.

- 카드뉴스: 1개 세트. 슬라이드 수 1~5장은 워커가 결정한다.
- 블로그: 본문 1개와 대표 이미지 1개
- 마케팅 소재: 같은 기획으로 결과 1~3개 선택

마케팅 소재를 2개 이상 생성할 때는 공통 생성 지시를 기본으로 사용하고, 원하는 경우 각 결과에 서로 다른 방향을 짧게 지정할 수 있다.

마케팅 소재를 여러 개 선택하면 성공한 결과 개수만큼 생성 사용량을 차감한다. 화면은 생성 시작 전에 예상 차감량과 남은 한도를 표시한다.

화면 우선 구현의 첨부 계약은 다음과 같다.

- 이미지: PNG, JPEG, WebP
- 문서: PDF
- 파일당 최대 10MB
- 전체 최대 5개
- 지원하지 않는 형식, 초과 크기와 중복 파일은 선택 즉시 안내한다.

### Step 6. 결과 확인

- 기획 중, 생성 중, 완료, 부분 실패, 실패 상태를 표시한다.
- 유형에 맞는 미리보기를 제공한다.
- 여러 결과를 생성한 경우 결과마다 상태, 실패 사유와 미리보기를 표시한다.
- 전체 재생성뿐 아니라 선택한 결과 또는 실패한 결과만 재생성할 수 있다.
- 재생성 시 간단한 재생성 사유를 입력할 수 있다.
- 개별 다운로드와 전체 다운로드를 제공한다.
- `게시 관리로 보내기`는 Instagram 연결이 완료된 카드뉴스에만 제공한다.
- Instagram이 연결되지 않았으면 버튼을 비활성화하고 `채널 연결 필요`와 이동 동작을 제공한다.
- 블로그와 마케팅 소재는 이번 범위에서 다운로드만 제공한다.
- 직접 편집 기능은 제공하지 않는다.

다운로드 결과 계약은 다음과 같다.

- 카드뉴스: PNG 이미지, 캡션·해시태그 TXT, 전체 ZIP
- 블로그: HTML, 대표 이미지, 전체 ZIP
- 마케팅 소재: 결과별 PNG와 카피 TXT, 선택 결과 또는 전체 ZIP
- 같은 결과를 다시 다운로드해도 신규 다운로드 사용량을 추가로 차감하지 않는다.

## 6. LLM 처리 설계

화면은 다음 비동기 처리 단계를 표현할 수 있어야 한다.

1. 제품 분석 LLM
   - 자사 정보, 선택한 제품 URL과 이미지를 바탕으로 사실, 특징, 타깃 후보, 소구점 후보를 생성한다.
   - 같은 입력 버전의 분석 결과는 재사용한다.
2. 사용자 선택
   - 타깃, 소구점, 트렌드 레퍼런스와 생성 지시를 확정한다.
3. 콘텐츠 기획 LLM
   - 핵심 메시지, 콘텐츠 구조, 카피와 시각 방향을 생성한다.
4. 최종 생성 LLM
   - 최종 본문, 캡션과 이미지 생성 프롬프트를 만든다.
5. 고정 이미지 모델
   - 카드뉴스, 대표 이미지와 광고 이미지를 생성한다.
6. 코드 검증
   - 필수 필드, 금지 표현, 근거 일치, 파일 규격을 확인한다.
   - 문제가 있는 부분만 한 번 재생성한다.

별도의 레퍼런스 분석 LLM 호출은 두지 않고 콘텐츠 기획 호출에 포함한다.

## 7. 전용 데이터 경계

AI 콘텐츠 생성은 자동 운영과 사용량·생성 목적이 다르므로 기존 `channel_outputs`에 직접 저장하지 않는다. 화면 우선 구현에서는 아래 계약을 mock 데이터로 제공하고, 후속 API·DB 단계에서 같은 경계를 유지한다.

- `ai_content_generations`: 생성 draft, 유형, 단계, 전체 상태와 생성 지시
- `ai_content_generation_outputs`: 한 생성 안의 개별 결과, 상태, 자산과 실패 사유
- `brand_audiences`: 브랜드에 저장한 타깃
- `brand_appeals`: 브랜드에 저장한 소구점
- `ai_content_generation_references`: 생성 당시 선택한 레퍼런스와 순서
- `ai_content_usage_ledger`: 생성·신규 다운로드 차감과 취소 원장

게시 관리로 보낸 카드뉴스만 기존 게시 도메인에 새 `channel_output`과 게시 큐를 생성하며, AI 콘텐츠 생성 결과 ID를 출처로 기록한다.

## 8. 사용량과 요금제 UI

AI 콘텐츠 생성 사용량은 자동 운영 사용량과 분리한다.

- 요금제별 `하루 생성 한도`와 `하루 신규 다운로드 한도`를 사용한다.
- 구체적인 한도 수치는 결제 금액 확정 후 설정한다.
- 화면 개발용 mock 값은 생성 5회, 신규 다운로드 10개를 사용한다.
- 사용량은 모든 생성 단계 상단에 계속 표시한다.
- 초기화 기준은 매일 00:00 KST다.
- 실패한 생성은 차감하지 않는다.
- 성공한 재생성은 생성 1회로 계산한다.
- 같은 결과물을 다시 다운로드하는 것은 추가 차감하지 않는다.
- 한도에 도달하면 신규 생성 또는 신규 결과물 다운로드 버튼을 비활성화한다.
- 한도 수치는 프론트에 하드코딩하지 않고 향후 API 응답으로 교체 가능한 mock 계약을 사용한다.

## 9. 화면 상태

공통 상태는 다음과 같다.

- 분석 대기
- 분석 중
- 입력 필요
- 생성 대기
- 기획 중
- 이미지 생성 중
- 완료
- 부분 실패
- 실패
- 한도 도달

단계별 입력은 브라우저 내 상태로 유지한다. 새로고침 영속화와 서버 임시 저장은 API 연결 단계에서 추가한다.

- 이전 단계로 돌아가도 입력과 선택 상태를 유지한다.
- 필수 입력이 없으면 `다음` 또는 `생성 시작`을 비활성화하고 누락 항목을 현재 화면에 표시한다.
- 생성 시작 전 화면을 벗어날 때 작성 중인 입력이 있으면 이동 확인을 표시한다.

## 10. 컴포넌트 경계

- `AiContentHomePage`: 유형, 사용량, 성과 우수 콘텐츠, 최근 결과
- `GenerationJobList`: 진행 중, 완료, 부분 실패와 실패 작업 목록
- `AiContentWizardPage`: 단계 이동과 전체 draft 상태
- `GenerationDetailPage`: 진행 상태와 결과 상세
- `UsageSummary`: 생성·다운로드 사용량
- `ContentTypeSelector`: 유형 선택
- `ProductAnalysisStep`: 제품·주제 입력과 분석 mock
- `ProductAnalysisDetail`: USP, 고객 문제, 구매 동기와 확인 근거 상세
- `AudienceAppealStep`: 타깃·소구점 선택
- `SavedAudienceAppealLibrary`: 브랜드에 저장한 타깃·소구점 탐색, 저장, 수정과 삭제
- `ReferencePickerStep`: 통합 레퍼런스 라이브러리와 최대 5개 선택 상태
- `ReferenceLibraryTabs`: 분야별, 소구점별, 직접 찾기와 내 보관함 전환
- `ReferenceCategoryFilters`: 대표·세부 분야, 형식과 출처 필터
- `ReferenceUploadDialog`: 마케팅 소재를 포함한 직접 레퍼런스 업로드
- `SelectedReferenceTray`: 선택한 레퍼런스의 미리보기, 제거와 순서 변경
- `GenerationBriefStep`: 생성 지시와 파일 첨부
- `GenerationResultPreview`: 유형별 결과 미리보기
- `GenerationOutputList`: 개별 결과 상태, 재생성과 다운로드

컴포넌트는 mock 데이터 소스를 API client와 분리해 후속 연결 시 화면을 다시 작성하지 않도록 한다.

결과 미리보기는 기존 `PublishArtifactPreview`의 이미지 갤러리, 이미지와 텍스트 표현을 재사용한다. AI 콘텐츠 생성 전용 화면은 결과 타입을 기존 `PublishArtifact` 계약으로 변환하며 별도의 미리보기 구현을 중복해서 만들지 않는다.

## 11. 오류·빈 상태

- 등록된 자사 정보 없음
- 분석 가능한 URL 또는 이미지 없음
- 저장한 트렌드 레퍼런스 없음
- 성과 데이터 없음
- 분석 실패
- 생성 부분 실패
- 생성 실패
- 오늘 생성 한도 도달
- 오늘 신규 다운로드 한도 도달

각 상태는 다음 행동을 명확히 제공한다. 자사 정보 등록, 트렌드 탐색 이동, 다시 분석, 재생성, 결제 및 구독 확인 중 하나다.

## 12. 화면 우선 구현 검증

- 세 경로가 인증된 AppShell 안에서 열린다.
- 사이드바에서 AI 콘텐츠 생성으로 이동할 수 있다.
- 세 콘텐츠 유형에서 생성 플로우를 시작할 수 있다.
- 6단계 이전·다음 이동과 필수 입력 차단이 동작한다.
- 분석, 생성, 완료, 실패, 한도 도달 mock 상태를 전환해 확인할 수 있다.
- 카드뉴스, 블로그, 마케팅 소재 결과 미리보기가 각각 렌더링된다.
- 진행 중, 완료, 부분 실패와 실패 생성 작업을 홈과 상세에서 확인할 수 있다.
- 저장한 타깃·소구점 재사용, 레퍼런스 순서 변경과 결과별 재생성 mock 동작이 가능하다.
- Instagram 연결 상태에 따라 게시 관리 전송 버튼의 활성화와 차단 사유가 달라진다.
- 데스크톱과 모바일에서 텍스트와 컨트롤이 겹치지 않는다.
- 모바일에서는 여섯 단계를 한 줄에 축소하지 않고 `현재 단계 / 전체 단계`와 단계명으로 표시한다.
- 키보드만으로 유형, 타깃, 소구점과 레퍼런스를 선택할 수 있고 선택 상태를 텍스트와 `aria-pressed`로 전달한다.
- 분석·생성 상태는 색상만으로 구분하지 않고 상태명과 진행 설명을 함께 제공한다.
- 기존 게시 관리, 트렌드 탐색, 브랜드 설정 화면 동작을 변경하지 않는다.
