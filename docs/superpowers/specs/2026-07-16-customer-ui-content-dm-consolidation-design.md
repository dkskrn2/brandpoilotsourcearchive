# 고객 UI 콘텐츠·DM 통합 설계

작성일: 2026-07-16  
상태: 확정  
범위: 고객용 React UI, 중앙 API, Instagram DM 수동응답, 콘텐츠 생성 프롬프트

## 1. 목표

분리된 콘텐츠 검토 흐름을 게시 관리로 통합하고, 자사 URL과 DM 운영 기능을 실제 사용 흐름에 맞게 재배치한다. 기존 크롤링, Wiki, 자동응답 상태 정책은 유지한다.

## 2. 콘텐츠 검토 통합

- 사이드바의 `콘텐츠 검토` 메뉴를 제거한다.
- `ContentPage`와 `/content` 독립 화면을 제거한다.
- 기존 `/content` 접근은 `/publish-queue?status=needs_review`로 이동시켜 오래된 링크가 깨지지 않게 한다.
- 게시 관리의 `검토 필요` 탭에서 기존 콘텐츠 검토 화면과 동일하게 승인, 거절, 재생성을 제공한다.
- 검토 대상의 콘텐츠가 생성된 경우 `완료` 항목에서 사용하는 상세 팝업과 같은 미디어 뷰어로 결과물을 확인한다.
- 팝업은 이미지, 캐러셀, 영상, HTML, 텍스트 결과물을 지원하고 내용이 길면 내부 스크롤한다.
- 결과물이 아직 생성되지 않은 항목은 미리보기 액션을 제공하지 않고 `콘텐츠 미생성` 상태만 표시한다.
- 검토 결과와 게시 상태는 기존 API 및 상태 모델을 사용한다.

## 3. 자사 URL 입력 위치

- 소스 화면에서 `자사 URL` 입력·관리 탭을 제거한다.
- 브랜드 설정 화면에 자사 URL 등록, 수정, 활성화·비활성화, 삭제, 재크롤링 기능을 이동한다.
- 참고 URL, 크롤링 이력, 주제표는 소스 화면에 유지한다.
- `source_urls`, 크롤링 API, 초기 크롤링, 중복 검사, URL 제한 정책은 변경하지 않는다.
- 온보딩의 자사 URL 안내 경로는 브랜드 설정으로 변경한다.

## 4. 전역 Top 버튼

- 공통 앱 레이아웃에 전역 Top 버튼을 한 번만 배치한다.
- 사용자가 일정 거리 이상 스크롤하면 우측 하단에 위쪽 화살표 아이콘 버튼을 표시한다.
- 클릭하면 문서 상단으로 부드럽게 이동한다.
- 로그인 화면을 제외한 모든 고객·관리 화면에서 동작한다.
- 모바일에서는 하단 주요 조작을 가리지 않는 여백을 적용한다.

## 5. DM 화면 재구성

- DM 자동답변 상단의 `확인 필요` 탭만 제거한다.
- 대화 목록 내부의 `확인 필요`, `불만`, `미답변`, `오류` 필터와 배지는 유지한다.
- 자동응답 중지, 확인 필요, 확인 완료 등 서버 상태값과 처리 정책은 변경하지 않는다.
- 데스크톱에서는 대화 목록을 상단에 고정하고 목록 영역만 독립 스크롤한다.
- 모바일에서는 기존 목록·대화 전환 방식을 유지하고 sticky를 적용하지 않는다.

## 6. DM 수동응답

- 선택한 대화 하단에 텍스트 입력창과 전송 버튼을 추가한다.
- 중앙 API가 기존 Instagram 메시지 전송 어댑터를 사용해 전송한다.
- 전송이 성공한 경우에만 발신 메시지를 DB에 저장하고 대화창에 반영한다.
- 수동응답은 자동응답 활성·중지 상태와 확인 필요 상태를 변경하지 않는다.
- 빈 메시지, 최대 길이 초과, 연결되지 않은 Instagram 계정, 권한 오류를 사용자에게 구분해 표시한다.
- 중복 클릭을 막고 전송 중에는 입력과 버튼을 잠근다.

## 7. 콘텐츠 참고 URL 비노출

- 대표 URL과 크롤링 원문은 워커에 계속 전달한다.
- Instagram, Threads 및 후속 채널 생성 프롬프트에 게시 본문, 캡션, 이미지 내 문구에 참고 URL을 출력하지 말라는 규칙을 추가한다.
- 운영 DB의 생성 근거, 내부 로그, 게시 관리의 운영 정보에는 URL을 유지한다.
- 저장 직전 URL 문자열 검사, 자동 삭제, URL 검출에 따른 생성 실패 처리는 추가하지 않는다.

## 8. Wiki 데이터와 확장

- Growthline 운영 DB 기준 FAQ 8개와 제품 3개가 활성 Wiki에 반영되어 있다.
- FAQ·제품 업로드는 `knowledge_entries` 저장 후 Wiki 재생성 요청을 자동 등록하는 현재 흐름을 유지한다.
- 이벤트 데이터는 이번 구현 범위에서 제외한다.
- 향후 이벤트를 추가할 때 `event` 지식 유형, 시작일, 종료일, 이벤트 URL, 활성 상태를 추가하고 기존 Wiki 수집·컴파일 흐름을 확장한다.

## 9. 오류 처리

- 콘텐츠 미생성과 미리보기 로딩 실패를 구분한다.
- 검토 저장 성공 후 목록 갱신 실패는 기존 성공 결과를 실패로 되돌리지 않는다.
- DM 수동응답의 모호한 전송 실패는 자동 재전송하지 않아 중복 메시지를 방지한다.
- API 일시 장애는 로그아웃으로 처리하지 않는다.

## 10. 테스트 범위

- 콘텐츠 검토 메뉴·라우트 제거 및 이전 URL 리다이렉트
- 게시 관리 검토 필요 탭의 결과물 유무별 팝업 노출
- 이미지, 영상, HTML, 텍스트 미리보기
- 자사 URL 관리 기능의 브랜드 설정 이동과 기존 API 호출 유지
- 전역 Top 버튼 표시와 스크롤 동작
- DM 상단 확인 필요 탭 제거와 대화 필터 유지
- DM 수동응답 성공, 입력 검증, 전송 실패, 중복 클릭 방지
- 콘텐츠 프롬프트의 참고 URL 비노출 규칙
- 기존 FAQ·제품 Wiki 빌드 회귀 테스트
