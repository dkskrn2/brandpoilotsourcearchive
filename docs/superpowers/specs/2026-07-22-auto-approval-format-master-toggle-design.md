# 자동 승인 형식 상위 토글 설계

## 목표

브랜드 설정의 `브랜드 전체 자동 승인`을 Instagram 콘텐츠 형식의 상위 제어로 사용한다. 상위 토글은 사용 가능한 카드뉴스, 릴스, 스토리를 한 번에 켜거나 끄며, 일부 형식만 선택된 상태를 명확히 표시한다. 이후 Threads와 TikTok 형식도 같은 계층에 추가할 수 있어야 한다.

## 사용자 동작

- 상위 토글을 ON으로 바꾸면 Meta 기능 상태가 `available`인 모든 하위 형식을 ON으로 바꾼다.
- 상위 토글을 OFF로 바꾸면 기능 상태와 관계없이 모든 하위 형식을 OFF로 바꾼다.
- `unchecked`, `unavailable`, `needs_attention` 형식은 상위 ON 시에도 OFF로 유지하고 해당 행에 이유를 표시한다.
- 사용자가 하위 형식을 개별적으로 끄면 상위 영역은 `부분 켜짐` 배지를 표시한다.
- 사용 가능한 모든 형식이 켜져 있으면 `전체 켜짐`, 모두 꺼져 있으면 `전체 꺼짐`을 표시한다.
- 부분 상태에서 상위 토글을 누르면 사용 가능한 모든 형식을 ON으로 복구한다. 전체 ON 상태에서 누르면 모두 OFF로 바꾼다.

## 기능 상태 판정

`brand_content_formats.capability_status`를 모든 Instagram 형식에 일관되게 사용한다.

- 공통 조건: Instagram 채널 연결, 외부 전문 계정 ID, 활성·미만료 자격 증명, 콘텐츠 게시 권한.
- 카드뉴스: 공통 조건을 충족하면 `available`.
- 릴스: 공통 조건을 충족하면 `available`. 영상 규격 오류는 개별 산출물 게시 오류로 처리하며 계정 기능 상태를 영구 차단하지 않는다.
- 스토리: 공통 조건과 기존 Story 게시 검증을 모두 충족해야 `available`.
- 토큰 만료, 권한 누락 또는 계정 연결 해제 시 관련 형식을 `needs_attention` 또는 `unchecked`로 바꾸고 자동으로 OFF 처리한다.

## 데이터 흐름

1. 브랜드 설정 진입 시 브랜드 프로필과 Instagram 형식 설정을 불러온다.
2. 화면은 형식별 `enabled`와 `capabilityStatus`로 전체/부분/꺼짐 상태를 계산한다.
3. 상위 토글 변경은 `autoApprovalEnabled`와 하위 `enabled` 초안을 함께 갱신한다.
4. 저장 시 기존 브랜드 프로필 API와 Instagram 형식 API에 변경된 값을 전송한다.
5. 서버는 사용할 수 없는 형식을 ON으로 저장하지 못하게 검증하고 최신 기능 상태를 응답한다.

## 오류 처리

- 프로필 또는 형식 저장 중 하나라도 실패하면 저장 완료로 표시하지 않고 기존 API 오류 안내를 보여준다.
- 서버가 기능 상태 변경으로 특정 형식을 거절하면 해당 형식을 OFF로 되돌리고 연결 또는 권한 확인 사유를 표시한다.
- 로딩 중에는 상위·하위 토글을 모두 비활성화해 중복 저장을 막는다.

## 테스트

- 상위 ON이 사용 가능한 형식만 모두 ON으로 바꾸는 UI 테스트.
- 상위 OFF가 모든 하위 형식을 OFF로 바꾸는 UI 테스트.
- 일부 형식 선택 시 `부분 켜짐`이 표시되는 테스트.
- 사용할 수 없는 형식이 상위 ON에도 OFF로 유지되는 테스트.
- 카드뉴스·릴스·스토리의 공통 기능 상태 판정 API 테스트.
- 저장 요청이 브랜드 전체 정책과 고정 형식 순서를 함께 보존하는 테스트.

## 범위 제외

- Threads와 TikTok 실제 형식 설정 추가.
- 릴스 영상 파일 자체의 사전 인코딩 검증.
- Meta 비즈니스 인증 및 앱 검수 자동화.
