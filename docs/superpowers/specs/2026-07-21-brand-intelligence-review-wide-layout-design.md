# 브랜드 정보 분석 결과 넓은 단일 열 UI 설계

## 목적

브랜드 정보 분석 결과 검토 화면이 데스크톱에서도 지나치게 좁게 표시되고, 긴 분석 문장이 textarea 내부 스크롤에 가려지는 문제를 해결한다. 사용자는 전체 결과를 위에서 아래로 읽으며 수정하고 저장할 수 있어야 한다.

## 확정안

- 분석 결과 검토 화면은 넓은 단일 열로 표시한다.
- 검토 영역은 공통 콘텐츠 영역을 모두 사용하되 최대 너비를 1180px로 제한한다.
- 기업 개요, 사업 소개, 핵심 타깃, 차별점, 핵심 소구점의 순서는 유지한다.
- 대표 분야와 직접 입력 세부 분야는 데스크톱에서만 2열로 유지하고 모바일에서는 1열로 전환한다.
- 경쟁사와 근거 영역도 분석 결과 영역과 동일한 폭을 사용한다.
- 긴 textarea는 내용에 맞춰 높이가 증가하며 자체 세로 스크롤을 만들지 않는다.
- textarea 수동 크기 조절은 제거하고 자동 높이 확장을 일관되게 적용한다.
- 저장 조건, API 요청, 분석 결과 데이터, 상태값은 변경하지 않는다.

## 반응형 동작

- 721px 이상: 검토 화면을 공통 콘텐츠 폭에 맞춰 최대 1180px로 표시한다.
- 720px 이하: 모든 필드를 단일 열로 표시하고 좌우 패딩을 유지한다.
- 긴 단어와 URL은 입력 영역을 넘지 않도록 줄바꿈한다.

## 접근성 및 상호작용

- 기존 label과 입력 요소 연결 구조를 유지한다.
- 키보드 입력과 포커스 순서는 현재 문서 순서를 따른다.
- textarea 자동 높이 계산은 최초 분석 결과 표시와 사용자 입력 변경 시 모두 적용한다.
- 저장 오류와 필수 입력 검증 메시지는 기존 위치와 의미를 유지한다.

## 테스트

- 분석 결과 검토 컴포넌트가 모든 필드를 기존 순서로 렌더링하는지 확인한다.
- 긴 초기 분석 결과를 가진 textarea의 높이가 내용에 맞게 갱신되는지 확인한다.
- 입력 변경 후 높이 재계산과 draft 업데이트가 함께 수행되는지 확인한다.
- 데스크톱 넓은 단일 열 및 모바일 단일 열 CSS 계약을 확인한다.
