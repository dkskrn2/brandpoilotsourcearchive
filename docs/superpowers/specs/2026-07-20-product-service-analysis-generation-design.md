# 제품·서비스 분석 기반 AI 콘텐츠 생성 설계

작성일: 2026-07-20
상태: 확정안
범위: AI 콘텐츠 스튜디오의 제품·서비스 분석, 타깃·소구점 선택, 레퍼런스 선택, CLI 생성 입력

## 1. 목적

사용자가 제품 또는 서비스 URL을 입력하면 상세 페이지의 사실과 이미지를 수집하고, 공개 웹 리서치를 결합해 타깃과 소구점을 추천한다. 사용자는 추천 결과와 레퍼런스, 프롬프트, 이미지를 선택한 뒤 카드뉴스·블로그·마케팅 소재를 생성한다.

핵심 원칙은 다음과 같다.

- URL 분석 결과는 같은 브랜드와 URL에서 재사용한다.
- 제품 페이지의 사실과 공개 웹 리서치를 구분해 저장한다.
- 공개 웹 리서치는 VOC·대안·시장 맥락에만 사용한다.
- 타깃은 3개를 추천하고 사용자는 정확히 1개를 선택한다.
- 선택한 타깃에 맞는 소구점을 추천하고 사용자는 정확히 1개를 선택한다.
- 제품 이미지 후보는 시스템이 대표 이미지를 자동 선택하며 사용자가 변경할 수 있다.
- 최종 콘텐츠 워커에는 화면의 임시 상태나 원문 전체가 아니라 검증된 구조화 스냅샷을 전달한다.

## 2. 생성 화면 흐름

### 2.1 콘텐츠 유형

- 카드뉴스
- 블로그
- 마케팅 소재

선택한 유형에 따라 마지막 생성 옵션과 실제 결과 계약만 달라진다. 제품·서비스 분석, 타깃·소구점, 레퍼런스 단계는 공통으로 사용한다.

### 2.2 제품·서비스 정보 및 분석

사용자는 `제품` 또는 `서비스`를 선택한다.

제품 입력:

- 제품 상세 URL
- 제품명과 프로모션 보완 입력
- 제품 이미지 후보
- 로고

서비스 입력:

- 서비스 소개 URL
- 서비스명
- 제공 내용, 이용 방식, 핵심 혜택의 직접 보완 입력
- 대표 이미지 또는 로고

URL 분석 순서:

1. 정규화 URL로 기존 분석을 조회한다.
2. 저장된 분석이 있으면 즉시 표시한다.
3. 저장된 분석이 없거나 사용자가 `다시 분석`을 선택하면 새 분석을 시작한다.
4. 중앙 크롤러가 제목, 설명, JSON-LD, Open Graph, 본문 사실과 이미지 후보를 추출한다.
5. 이미지 후보를 Object Storage에 복사하고 크기, 비율, 대체 텍스트, 출처 URL을 저장한다.
6. 분석 워커가 이미지와 제품 사실을 검토해 대표 이미지 1개를 자동 선택한다.
7. 사용자는 다른 후보로 변경하거나 직접 이미지를 추가할 수 있다.

공개 웹 검색에서 발견한 이미지는 자동으로 저장하거나 콘텐츠에 사용하지 않는다. 제품·서비스 입력 URL에서 추출했거나 사용자가 직접 올린 이미지만 생성 입력으로 사용할 수 있다.

### 2.3 제품·시장 분석

분석 화면은 다음 결과를 제공한다.

- 목적·상황·니즈
- 대안 3개의 장점과 한계
- 공개 커뮤니티 기반 VOC
- 제품·서비스 USP와 근거
- 고효율·고전환 타깃 세그먼트 3개

타깃 계약:

- 타깃명
- 특징
- Pain Point
- 구매 동기
- 제품 USP와 연결되는 근거
- 근거 출처

시스템은 가장 근거가 강한 타깃 1개를 기본 선택하고 나머지 2개를 대안으로 표시한다. 사용자는 타깃을 변경하거나 직접 추가·수정할 수 있다.

### 2.4 소구점 선택

선택한 타깃을 기준으로 소구점 후보를 생성한다. 사용자는 정확히 1개의 소구점만 선택한다. 핵심·보조 구분과 복수 선택은 제공하지 않는다.

소구점 계약:

- 제목
- 설명
- 근거 유형
- 선택한 타깃과의 연결 이유
- 근거 출처

선택하거나 직접 작성한 타깃과 소구점은 기존 `brand_audiences`, `brand_appeals` 재사용 목록에 저장할 수 있다.

### 2.5 레퍼런스 선택

기존 레퍼런스 선택 기능을 유지한다.

- 업종별
- 콘셉트별
- 직접 찾기
- 내 보관함
- 직접 파일 업로드
- 선택 순서 변경

레퍼런스는 표현 방식과 시각적 방향을 참고하는 자료다. 제품 사실의 근거로 사용하지 않는다.

### 2.6 프롬프트 및 생성 옵션

이 단계가 마지막 단계이며 별도의 최종 확인 화면은 두지 않는다.

- 결과별 프롬프트
- 여러 결과에 프롬프트 전체 적용
- 디자인 색상
- 제품 이미지 선택·추가
- 인물 이미지 선택·추가
- 크기·비율 참고 이미지
- 제작 개수
- 콘텐츠 생성 버튼

디자인 색상은 브랜드 설정의 대표 색상을 기본값으로 채운다. 사용자가 직접 수정할 수 있지만 레퍼런스에서 색상을 자동 추출하지 않는다.

제품 분석에서 저장한 이미지는 분석 재사용을 위해 보관한다. 마지막 단계에서 사용자가 직접 첨부한 인물·크기·시각 참고 이미지는 생성 완료 후 삭제한다.

## 3. 데이터 모델

### 3.1 `ai_content_subject_analyses`

제품·서비스 URL별 재사용 가능한 분석이다.

- `id`, `workspace_id`, `brand_id`
- `subject_type`: `product | service`
- `source_url`, `normalized_url`
- `input_json`: 사용자가 보완한 제품명, 프로모션, 서비스 설명
- `status`: `queued | extracting | researching | ready | partial | failed`
- `facts_json`: 입력 URL에서 확인한 사실과 출처
- `research_json`: VOC, 대안, 시장 맥락과 출처
- `targets_json`: 추천 타깃 3개
- `appeals_json`: 타깃별 소구점 후보
- `selected_image_id`
- `analysis_version`
- `error_code`, `error_message`
- `created_at`, `updated_at`, `completed_at`

활성 분석은 `brand_id + subject_type + normalized_url` 기준으로 하나만 유지한다. 자동 만료시키지 않고 사용자가 `다시 분석`할 때 새 버전으로 교체한다.

### 3.2 `ai_content_subject_images`

- `id`, `analysis_id`, `workspace_id`, `brand_id`
- `source_url`, `storage_url`, `storage_path`
- `width`, `height`, `mime_type`, `alt_text`
- `role`: `product | service | logo | detail | unknown`
- `selection_score`, `is_selected`
- `created_at`, `deleted_at`

이미지 원본 URL의 만료와 핫링크 차단에 대비해 허용된 이미지는 Object Storage에 복사한다. 이미지 크기와 응답 제한, SSRF 방어는 기존 크롤러 정책을 적용한다.

### 3.3 generation 연결

`ai_content_generations.draft_json`에는 분석 원본을 중복 저장하지 않고 다음 스냅샷을 기록한다.

- `subjectAnalysisId`, `subjectAnalysisVersion`
- 선택한 이미지 ID와 저장 URL
- 선택한 타깃 스냅샷
- 선택한 소구점 스냅샷
- 레퍼런스 ID와 순서
- 최종 프롬프트와 생성 옵션

과거 결과를 재현할 수 있도록 생성 시작 시점의 값은 generation과 reference snapshot에 고정한다.

## 4. CLI 작업 분리

### 4.1 제품·서비스 분석 워커

분석 워커만 공개 웹 검색을 수행한다. 중앙 API는 아래 `subject-analysis.v1` 입력을 전달한다.

```json
{
  "contractVersion": "subject-analysis.v1",
  "brand": {
    "name": "브랜드명",
    "primaryCategory": "대표 분야",
    "subcategories": ["세부 분야"],
    "brandColor": "#0057B8"
  },
  "subject": {
    "type": "product",
    "sourceUrl": "https://example.com/product",
    "manualInput": {
      "name": "제품명",
      "promotion": "프로모션",
      "description": "사용자 보완 설명"
    }
  },
  "extracted": {
    "facts": [],
    "structuredData": {},
    "imageCandidates": []
  },
  "researchPolicy": {
    "publicWebSearch": true,
    "allowedPurposes": ["voc", "alternatives", "market_context"],
    "requireSourceUrl": true
  }
}
```

워커 출력은 `subject-analysis-result.v1`로 고정한다.

```json
{
  "contractVersion": "subject-analysis-result.v1",
  "summary": "분석 요약",
  "needs": [],
  "alternatives": [],
  "voc": [],
  "usps": [],
  "targets": [],
  "appealsByTarget": {},
  "recommendedImageId": "uuid",
  "sourceGaps": []
}
```

확인되지 않은 가격, 효과, 후기, 수치와 경험은 생성하지 않는다. 출처가 없는 공개 웹 주장은 결과에서 제외하고 `sourceGaps`에 기록한다.

### 4.2 최종 품질 분석 워커

사용자가 마지막 단계에서 생성을 누른 뒤 기존 `content-quality.v1` 분석을 수행한다. 이 워커는 웹 검색을 반복하지 않고 확정 스냅샷을 이용한다.

전달 데이터:

- 브랜드 Wiki 스냅샷
- 제품·서비스 분석 스냅샷과 출처
- 선택한 제품 이미지
- 선택한 타깃 1개
- 선택한 소구점 1개
- 선택한 레퍼런스 스냅샷
- 사용자 프롬프트
- 색상, 비율, 제작 개수
- 사용자 첨부 이미지

최종 분석은 콘텐츠의 훅, 독자 효익, 구체적 주장, 근거와 누락 정보를 결정한다.

### 4.3 유형별 콘텐츠 워커

카드뉴스, 블로그, 마케팅 소재 워커는 공통 `content-generation-input.v2` 봉투를 받는다.

```json
{
  "contractVersion": "content-generation-input.v2",
  "contentType": "card_news",
  "brandContext": {},
  "subject": {
    "facts": [],
    "research": {},
    "selectedImages": []
  },
  "message": {
    "target": {},
    "appeal": {},
    "qualityBrief": {}
  },
  "creativeDirection": {
    "prompt": "사용자 입력",
    "brandColor": "#0057B8",
    "selectedColor": "#0057B8",
    "aspectRatio": "1:1",
    "outputCount": 1
  },
  "references": [],
  "attachments": []
}
```

유형별 워커는 공통 봉투에 추가 규칙만 적용한다.

- 카드뉴스: 1~5장, 슬라이드별 정보 구조, 캡션과 해시태그
- 블로그: SEO 제목·메타·HTML, 설명에 필요한 본문 이미지
- 마케팅 소재: 결과별 프롬프트와 메시지 가설, 단일 이미지와 카피

분석 워커의 리서치 결과와 최종 생성 워커의 표현 책임을 분리해 같은 제품 분석을 여러 유형에서 재사용한다.

## 5. 상태와 오류 처리

- 캐시 조회 중: 저장된 분석 확인
- 페이지 추출 중: 제품·서비스 사실과 이미지 수집
- 공개 웹 조사 중: VOC·대안·타깃 분석
- 부분 완료: 제품 사실은 있으나 공개 웹 근거가 부족함
- 완료: 이미지, 타깃, 소구점 선택 가능
- 실패: 원인 표시 후 다시 분석 또는 직접 입력

URL을 읽지 못하면 제품 사실을 추측하지 않는다. 서비스는 직접 설명으로 계속 진행할 수 있다. 제품 이미지가 없으면 사용자가 이미지를 올려야 다음 단계로 진행할 수 있다. 공개 웹 근거가 부족해도 제품 페이지 사실이 충분하면 부분 완료로 진행할 수 있다.

분석 요청과 생성 요청은 각각 idempotency key를 사용한다. 분석 CLI의 결과가 계약을 위반하면 제한된 횟수만 재시도하고, 이전에 저장된 정상 분석은 덮어쓰지 않는다.

## 6. 검증 범위

- 같은 정규화 URL에서 저장 분석을 즉시 재사용하는지
- `다시 분석`에서만 새 분석 버전이 생성되는지
- 제품 페이지 이미지 후보와 대표 이미지 선택이 브랜드 범위를 벗어나지 않는지
- 추천 타깃이 정확히 3개이고 선택 타깃은 1개인지
- 소구점이 정확히 1개만 선택되는지
- 공개 웹 주장에 출처가 없으면 제외되는지
- 브랜드 색상이 기본값이고 사용자 변경값이 최종 워커에 전달되는지
- 제품 이미지와 임시 사용자 첨부의 삭제 정책이 구분되는지
- 카드뉴스·블로그·마케팅 워커가 같은 입력 스냅샷에서 각 계약의 산출물을 만드는지
- 재시도와 중복 클릭에서 작업과 사용량이 중복 생성되지 않는지

## 7. 제외 범위

- 생성 결과 직접 편집기
- 레퍼런스 이미지 색상 자동 추출
- 공개 웹에서 찾은 타사 이미지를 자동 다운로드하거나 재사용하는 기능
- 분석 결과 자동 만료와 주기적 재분석
- 영상 생성
