# 게시관리 정방형 카드 UI 설계

작성일: 2026-07-19  
상태: 확정  
범위: 고객용 게시관리 화면 UI

## 1. 목표

게시관리의 기존 테이블을 정방형 카드 목록으로 바꿔 콘텐츠 상태와 결과물을 빠르게 비교할 수 있게 한다.

이번 변경은 표시 방식만 바꾼다. API, DB 상태값, 검토·승인·재생성·거절, 게시 큐와 상세 팝업 동작은 변경하지 않는다.

## 2. 상태 필터

내부 상태값은 유지하고 화면에서는 다음 다섯 상태 그룹과 전체 필터만 제공한다.

| 화면 필터 | 포함하는 현재 상태 |
|---|---|
| 전체 | 모든 상태 |
| 준비 중 | `generating`, `queued` |
| 검토 필요 | `needs_review` |
| 게시 예정 | `publish_queued`, `scheduled`, `publishing` |
| 완료 | `completed` |
| 문제 | `failed`, `rejected` |

현재 `대기`는 콘텐츠 결과가 아직 준비되지 않은 주제·생성 대기 상태다. `게시 대기`는 콘텐츠 생성과 승인이 끝나 게시 큐에 들어간 상태다. 기능상 다른 단계이므로 내부 상태는 합치지 않고, 사용자에게는 각각 `준비 중`과 `게시 예정` 그룹으로 보여준다.

필터에는 현재 건수를 함께 표시한다. 결과가 0건인 필터도 화면 구조가 흔들리지 않도록 유지한다.

## 3. 카드 구조

모든 목록 카드는 정방형으로 통일한다.

카드에는 다음 정보만 표시한다.

1. 결과물 미리보기
2. 콘텐츠 제목
3. 통합 상태 배지
4. 채널과 결과 형식
5. 게시 예정 또는 완료 일시
6. 현재 상태에서 가능한 핵심 작업

소스 URL, 상세 생성 근거, 채널별 오류 전문과 전체 게시 정보는 카드에 펼치지 않고 기존 상세 팝업에서 확인한다.

## 4. 미디어 표시

정방형 카드 내부에 고정 미리보기 영역을 둔다.

- 정방형 이미지: 미리보기 영역을 채워 표시
- 세로 이미지·영상: 원본 비율을 유지해 중앙에 전체 표시
- 가로 이미지·영상: 원본 비율을 유지해 중앙에 전체 표시
- HTML: 대표 이미지가 있으면 대표 이미지를 표시하고, 없으면 문서 미리보기 상태를 표시
- 텍스트: 제목과 짧은 본문 미리보기 표시
- 결과 미생성: 생성 전 또는 생성 중 상태 표시
- 결과 생성 실패: 실패 상태와 상세 확인 동작 표시

이미지와 영상은 `object-fit: contain`을 사용하며 크롭하지 않는다. 실제 해상도와 전체 결과는 상세 팝업에서 확인한다.

## 5. 카드 작업

카드 전체 또는 `콘텐츠 보기`를 누르면 기존 콘텐츠 상세 팝업을 연다.

상태별 핵심 작업은 기존 로직을 그대로 사용한다.

- 검토 필요: 콘텐츠 보기, 승인, 재생성, 거절
- 준비 중: 현재 생성 단계 표시, 작업 버튼 비활성화
- 게시 예정: 콘텐츠 보기, 예약 시각 확인
- 완료: 콘텐츠 보기, 외부 게시물 열기, 허용된 경우 다운로드
- 문제: 콘텐츠 보기, 오류 확인, 기존 재시도 동작

채널이 여러 개인 주제는 한 카드 안에 채널·형식 요약을 표시하고 상세 팝업에서 채널별 결과를 확인한다.

## 6. 반응형

- 넓은 데스크톱: 3열
- 태블릿과 좁은 데스크톱: 2열
- 모바일: 1열

카드의 정방형 비율은 모든 화면에서 유지한다. 긴 제목은 두 줄로 제한하고 넘치는 내용은 상세 팝업에서 확인한다. 버튼과 상태 배지가 미리보기 영역을 가리지 않도록 미디어와 정보 영역을 분리한다.

## 7. 빈 상태와 로딩

- 최초 로딩: 정방형 카드 스켈레톤을 현재 열 수만큼 표시
- 필터 결과 없음: 선택한 상태에 해당하는 콘텐츠가 없다는 빈 상태 표시
- API 오류: 기존 오류 안내와 다시 시도 동작 유지
- 일부 미디어 로딩 실패: 카드 정보는 유지하고 미리보기 영역에 불러오기 실패 표시

## 8. 접근성

- 카드 상태는 색상뿐 아니라 텍스트로 표시
- 미디어에는 콘텐츠 제목과 형식을 포함한 대체 텍스트 제공
- 카드 작업 버튼은 키보드로 접근 가능
- 카드 전체 클릭과 내부 버튼 동작이 충돌하지 않도록 상세 열기 영역을 명확히 구분
- 필터는 현재 선택 상태를 `aria-pressed`로 제공

## 9. 구현 경계

이번 범위에 포함한다.

- 기존 테이블을 정방형 카드 그리드로 교체
- 화면 필터를 여섯 항목으로 재구성
- 기존 상태를 화면 필터 그룹으로 매핑
- 카드형 로딩·빈 상태·반응형 스타일
- 기존 상세 팝업과 검토 작업 연결

이번 범위에서 제외한다.

- API 또는 DB 상태값 변경
- 게시 스케줄 정책 변경
- 승인 및 자동 승인 정책 변경
- 미디어 파일 변환 또는 크롭
- 새로운 게시·다운로드 기능 추가

## 10. 검증 기준

1. 정방형·세로형·가로형·HTML·텍스트 결과가 같은 크기의 카드에 표시된다.
2. 모든 미디어가 크롭 없이 전체 표시된다.
3. 기존 상태가 정해진 화면 필터 그룹에 빠짐없이 포함된다.
4. 기존 콘텐츠 상세, 승인, 재생성, 거절과 게시 결과 동작이 유지된다.
5. 데스크톱, 태블릿, 모바일에서 각각 3열, 2열, 1열로 표시된다.
6. 필터·카드·상세 팝업을 키보드로 사용할 수 있다.
