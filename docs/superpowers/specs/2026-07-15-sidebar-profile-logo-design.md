# Sidebar Brand Profile And Logo Design

작성일: 2026-07-15  
상태: Approved for implementation planning

## 1. 목표

고객용 앱의 사이드바 하단에 현재 브랜드를 식별할 수 있는 로고와 브랜드명을 표시한다. 브랜드 설정 화면에서는 같은 로고를 등록, 변경, 삭제할 수 있게 한다.

## 2. 확정 UI

- 비교 시안의 A안인 컴팩트 통합형을 사용한다.
- 사이드바는 세로 flex 레이아웃을 유지하고 프로필 영역을 하단에 고정한다.
- 프로필 영역에는 원형 로고, 브랜드명, `브랜드 설정` 보조 문구, 이동 표시 아이콘을 배치한다.
- 프로필 영역 전체를 클릭하면 `/brand-settings`로 바로 이동한다.
- 별도 프로필 메뉴는 만들지 않는다. 로그아웃은 기존 상단 메뉴에 유지한다.
- 로고가 없거나 로드에 실패하면 브랜드명의 첫 글자 1~2자를 대체 표시한다.
- 긴 브랜드명은 한 줄 말줄임 처리한다.

## 3. 브랜드 설정

- 기존 `브랜드 프로필` 패널 왼쪽에 로고 편집 영역을 통합한다.
- 로고 미리보기 아래에 `이미지 등록` 또는 `이미지 변경` 버튼을 둔다.
- 등록된 로고가 있을 때만 삭제 버튼을 제공한다.
- 허용 형식은 PNG, JPEG, WebP이며 최대 크기는 2MB다.
- 정사각형 이미지를 권장하지만 원본 파일을 자르거나 변환하지 않는다.
- 사이드바에서는 `object-fit: cover`로 원형 프레임 안에 표시한다.
- 로고 업로드와 삭제는 선택 즉시 별도 API로 처리한다. 기존 페이지의 텍스트 설정 저장 버튼과 섞지 않는다.

## 4. 데이터와 저장소

- `brand_profiles`에 `logo_url`과 `logo_storage_path` nullable 컬럼을 추가한다.
- 비민감 공개 자산인 로고는 Supabase Storage의 public `brand-assets` 버킷에 저장한다.
- 저장 경로는 `{workspaceId}/{brandId}/logo-{uuid}.{extension}` 형식을 사용한다.
- 새 로고 저장과 DB 갱신이 성공한 뒤 기존 로고 객체를 삭제한다.
- 삭제 실패는 로그에 남기되 새 로고 저장을 되돌리지 않는다.

## 5. API

- `GET /brands/:brandId/profile` 응답에 `logoUrl`을 추가한다.
- `POST /brands/:brandId/logo`는 `fileName`, `mimeType`, `fileBase64`를 받아 형식과 용량을 검증한 후 저장한다.
- `DELETE /brands/:brandId/logo`는 Storage 객체를 삭제하고 프로필의 로고 필드를 비운다.
- 인증 및 워크스페이스 소유권 검사는 기존 브랜드 설정 API와 동일하게 적용한다.
- `GET /brands/:brandId/ui-status`에도 `logoUrl`을 포함해 사이드바가 별도 프로필 요청 없이 렌더링되게 한다.

## 6. 상태와 오류 처리

- 업로드 중에는 로고 버튼을 비활성화하고 진행 상태를 표시한다.
- 형식 오류와 2MB 초과는 API 호출 전에 안내하고 서버에서도 다시 검증한다.
- 업로드 실패 시 기존 로고를 유지하고 오류 안내를 표시한다.
- 이미지 로드 실패 시 사이드바와 설정 화면 모두 글자 대체 로고로 전환한다.
- Supabase Storage 환경 설정이 없으면 `brand_logo_storage_not_configured` 오류를 반환한다.

## 7. 컴포넌트 경계

- `BrandLogo`: 로고 URL과 브랜드명을 받아 이미지 또는 글자 대체 로고를 렌더링한다.
- `SidebarBrandProfile`: 브랜드 상태를 표시하고 브랜드 설정으로 이동한다.
- `BrandLogoEditor`: 파일 선택, 미리보기, 업로드, 삭제 상태를 관리한다.
- `BrandSettingsPage`: 기존 프로필 폼과 `BrandLogoEditor`를 조합하며 로고 업로드 내부 구현은 갖지 않는다.

## 8. 테스트

- API: 정상 업로드, 지원하지 않는 MIME, 용량 초과, 삭제, 소유권 거부를 검증한다.
- Repository: 프로필 및 UI 상태에 `logoUrl`이 매핑되는지 검증한다.
- Frontend: 로고 표시, 대체 글자, 직접 이동, 업로드 실패 시 기존 로고 유지, 삭제를 검증한다.
- 반응형: 데스크톱 사이드바와 모바일 레이아웃에서 정보가 겹치지 않는지 확인한다.

## 9. 제외 범위

- 사용자 개인 프로필 사진
- 브랜드 전환 메뉴
- 사이드바 프로필 팝업
- 이미지 자르기 및 편집기
- 비공개 signed URL 운영
